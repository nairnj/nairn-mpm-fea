FEA Boundary Conditions

FEA calculations require mesh-based boundary conditions and their details are given on the OSUPDocs wiki. You can use commands to generate the boundary conditions and/or commands that explicitly list. All boundary conditions, along with a few other options, must be within the single <GridBCs> block: 

<GridBCs>
   (any number of boundary condition commands)
   <Cracktip x='0' y='0'/>
   
   <Periodic dof='x' delta='1' slope='0'/>
   <Periodic dof='y' delta='1' shear='0'/>

   <Resequence x='0' y='0'/>
</GridBCs>

Generated Boundary Condition Definition

Most oundary conditions can be generated with <BCLine> and <BCPt>: 

  • <BCLine x1='0' y1='10' x2='0' y2='20' tolerance='.01'>
  (one of more boundary condition command)
</BCLine>
    Assign boundary condition at nodes along a defined line from (x1,y1) to (x2,y2) (in length units) or along a previously defined path (in path attribute). The optional tolerance finds all nodes within that distance of the line.
  • <BCLine path='(id)' select='1'>
  (one of more boundary condition command)
</BCLine>
    Assign boundary condition at nodes along a previously defined path. An optional select='1' attribute will select all nodes along the line or path for conditional output.
  • <BCPt x='0' y='10' select='1'>
  (one of more boundary condition command)
</BCPt>
    Assign boundary conditions to the node closed to the point (x,y) (in length units). An optional select='1' attribute can be used to select the node for conditional output.
  • <BCPt keypt='(id)' select='1'>
  (one of more boundary condition command)
</BCPt>
    Assign boundary conditions to the node at a previously defined keypoint. An optional select='1' attribute can be used to select the node for conditional output.

Subordinate Boundary Conditions Commands

Commands subordinate to any of the above shape commands can set specific boundary conditions. The possible commands are: 

  • <DisBC dof='(dirnum)' disp='(disp)'/>
<DisBC dof='(dirnum)' function='(function)'/>
    Set fixed displacement (in length units) to all nodes defined by the shape in direction (dirnum), which is 1 or 2 for x or y direction. The (disp) options sets a numerical displacement while the (function) sets it with a user-defined function.
  • <LoadBC dof='(dirnum)' load='(load)'/>
<LoadBC dof='(dirnum)' function='(load)'/>
    Set fixed load (in force units) to all nodes defined by the shape in direction (dirnum), which is 1 or 2 for x or y direction. The (load) option sets a numerical load while the (function) sets it with a user-defined function.
  • <StressBC dir='(dirnum)' stress='(stress1),(stress2),(stress3)'/>
    Apply stress (in pressure units) along the path defined by the shape in direction (dirnum), which is 1 or normal and 2 for shear. There can be 1 to 3 comma delimited stresses. One gives a constant stress, two gives linear variation in stress between path end points, and three allows a quadratic variation in stress through those three values. This condition is only allowed in <BCLine> that selects a path.
  • <rotate axis='(axis)' angle='(ange)'/>
    Rotates the degrees of freedom at all nodes defined by the shape. The rotation (axis) must be 3 for 2D problems; (angle) is the clockwise rotation in degrees. Once node(s) are rotated, you can specify displacement boundary conditions in the rotated coordinates. Rotation only affects displacement boundary conditions.

Explicit Boundary Condition Definition

Another way to define nodal displacement, nodal load, or element edge stress boundary conditions is to explicit list each one. This method is done by defining three block elements with the <GridBCs> block for displacement conditions (<DisplacementBCs>) for nodal loade (<LoadBCs>), for element edge stresses (<EdgeBCs>). Subordinate to each block are commands to apply a boundary conditions to one node, one node, or one edge:

<DisplacementBCs>
  (one or more <fix> or <rotate> commands)
</DisplacementBCs>
   
<LoadBCs>
  (one or more <load> commands)
</LoadBCs>
   
<EdgeBCs>
     <stress elem='3' face='1' dir='1'>0,100</load>
</EdgeBCs>

The following commands apply a boundary condition to one node or one element edge:

  • <fix node='24' dof='2'>0</fix>
    Set displacement on one node in direction 1 or 2 for x or y direction (using dof attribute). The value of the command is the displacement (in length units).
  • <rotate node='5' axis='3'>45</rotate>
    Rotates the degrees of freedom one node. The rotation (axis) must be 3 for 2D problems; the value of the command is the clockwise rotation in degrees. Once a node is rotated, you can specify displacement boundary conditions in the rotated coordinates.
  • <load node='5' dof='1'>45</load>
    This commands specifies a node (by number) and the direction of the load with dof of 1 or 2 for x or y direction. The value of the element is the applied load (in force units).
  • <stress elem='3' face='1' dir='1'>0,100</load>
    This command specifies an element (using elem), a face of the element, and a direction (using dir with setting 1 or n for normal stress or 2 or t for tangential or shear stress). The value of the <stress> tag should be 1 to 3 stresses (in pressure units), separated by commas. One gives a constant stress, two gives linear variation in stress between path end points, and three allows a quadratic variation in stress through those three values.

Periodic Boundary Conditions

<Periodic> commands with the <GridBCs> block can be used to run calculations that are periodic in strain in just the x direction, just the y direction, or in both the x and y directions. Periodic simulations are decribed in detail on the OSUPDocs wiki. the command is:

<Periodic dof='(dir)' delta='(delta)' slope='(slope)'/>

where

  • (dir) is x (or 1) or y (or 2) or Z (or 3) to define the periodic direction. The Z option is for axisymmetric calculations and the only one allowed in such calculations
  • The delta, slope (or shear) attributes let you specify an average displacement jump (in length units) and rotation (dimensionless) or shear (in length units) between the two ends in the periodic direction(s); it is a rotation when only one direction is periodic or shear when both are periodic. The results are summarized in the table below.

The following table summarizes the periodic options where Δx, Δy, and ΔZ are the length of the body in the x, y, and Z directions.

Resequencing the Nodes

Finite element analysis is more efficient if the nodes are numbered to minimize bandwidth of the problem. These commands will resequence the nodes for minimum bandwidth:

<Resequence x='0' y='0'/>
<Resequence keypt='(id)'/>

The resequencing will start at the node closest to the (x,y) coordinates provided in the x and y attributes (in length units) or at the key point with specified (id). It is best to resequence from a node on a corner of the object.
