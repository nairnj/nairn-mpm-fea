Scheduling Custom Tasks

NairnMPM supports custom tasks that are performed at the end of each time step (for more details see the OSUPDocs Wiki). The tasks can range from simple calculations to iterations over all material points and nodes. New custom tasks can be written by users working with NairnMPM source code and then scheduled when needed.

The format for scheduling any available custom task is as follows:

<CustomTasks>
  <Schedule name='(taskName)'>
    <Parameter name='(paramName)'>(value)</Parameter>
         ⋮
  </Schedule>
         ⋮
</CustomTasks>

All custom tasks are scheduled within a single <CustomTasks> block. A specific task is scheduled with a <Schedule> command where the name attribute specifies the task name. Any task may depend on parameters which can be set with a series of <Parameter> commands. The name attribute gives the name of the parameter and the element contents (in (value)) is the setting for that parameter (although not all parameters required a value).

The currently available (and documented) custom tasks are their parameters are:

VTKArchive
This task allows NairnMPM to export data extrapolated to the grid during the calculation to VTK Legacy files (these files can be visualized by other software such as Paraview). The parameters (by name) define what is saved in the files:
  • mass - nodal mass (in mass units)
  • material - the material number to visualize different materials in a composite.
  • displacement - the displacement vector (in length units).
  • velocity - the velocity vector (in velocity units).
  • stress - the stress tensor (in pressure units).
  • pressure - the pressure (in pressure units).
  • equivstress - the equivalent stress or von Mises stress (in pressure units).
  • strain - the strain tensor, which is elastic strain for materials that partition elastic and plastic strain, but total strain for all others (absolute).
  • deltav - the relative volume change (V-V0)/V0 (absolute).
  • equivstrain - the equivalent strain from above strain (absolute).
  • plasticstrain - the plastic strain tensor (absolute). Some materials have plastic strain, but hyperelastic materials have left Cauchy Green strain in the particle's plastic strain.
  • totalstrain - Archive total strain tensor (absolute, elastic+plastic strain for materials that separate them).
  • temperature - the temperature (in K).
  • concentration - the concentration (in wt %).
  • strainenergy - the strain energy in (in energy units).
  • heatenergy - the heat energy (in energy units).
  • plasticenergy - the plastic energy (in energy units).
  • contactforce - the nodal force on the grid for MultiMaterialMode simulations when they include rigid materials that have SetDirection=8. It is force of the rigid material on the object in (in force units).
  • volumegradient (matnum) - the volume gradient used for contact calculations when running in multimaterial model. The (matnum) argument is the material number whose gradient is archived.
  • numpoints - the number of material points interacting with each node in the grid.
  • archiveTime (time) - enter the time interval (in alt time units) between saving of VTK archives. If this parameter is omitted, the VTK archive files are written on the same steps as the particle archives.
  • firstArchiveTime (time) - enter the time to save the first results (in alt time units).
HistoryArchive
This custom task provides a method to archive more history variables and also provides an alternate format for archiving history variables 1 through 4. When this task is activated, the history data on each particle will be written to tab-delimited text files in the output results folder. You select which history variables to archive, and their order, using a series of Parameter commands: 
   <Parameter name='(num)'/>
where (num) is the history variable number to be saved. Two other parameters are:
  • archiveTime - enter the time interval (in alt time units) between saving of history archives. If this parameter is omitted, the history archive files are written on the same steps as the particle archives.
  • firstArchiveTime - enter the time to save the first history results (in alt time units).
ReverseLoad
This custom task monitors crack lengths or any currently archived global quantity. To monitor crack lengths, enter the following parameters:
  • maxLength (length) - this parameter specifies the maximum crack length (in length units). Once this length is reached, the specified task action is triggered.
  • crackNumber (num) - this parameter gives the crack number to watch. When that crack reaches the specified maximum length, the task action will be triggered. Alternatively, crackNumber can be zero which causes the task action to trigger when any crack reaches the specified maximum length. The default value is 0 (or any crack).
Alternatively, to trigger task action on a global quantity, enter the following parameters:
  • "global (quantity)" - specify the global archive quantity within the parameter name (e.g., "global sxx") and the parameter takes no value. The use of this parameter converts the task to trigger on critical value of a global quantity rather then a crack length.
  • mat (num) - if the global archive being used as a trigger is for a specific material or a boundary condition ID, enter that material (by number greater than zero) or the boundary condition ID (number less than zero) in this parameter.
  • maxValue (value) - this parameter specifies the critical value for the global quantity. If the value is positive, the trigger happens when the quantity first exceeds this value; if it is negative, the trigger happens when the quantity becomes more negative than this value.
When this task is triggered by critical crack length, the crack propagation will stop and then the task action will take affect. When the trigger is a global quantity, reaching the critical value will trigger the action (but crack propagation will be unaffected). The possible task actions after the trigger are set with the following parameters:
  • style (mode) - the (mode) option determines what happens after the task is triggered. The default value is 0. The options are:
    • 0: Reverses all linearly increasing particle loads, particle tractions, and constant-velocity rigid particles and stop when they return to zero.
    • 1: Stops all linearly increasing particle loads and tractions at their current value and zeros the velocity of all constant-velocity rigid particles. The analysis continues.
    • 2: All particle loads and tractions and rigid particles continue unchanged. Only the crack propagation stops. This option is meaningless and therefore not allowed when the task is triggered on a global quantity.
    • 3: The analysis terminates.
  • hold (holdTime) - when (holdTime) is positive, the task will hold at current boundary conditions for the specified time (in alt time units) and then proceed with the style option. A hold phase is only allowed when style is 0 (to reverse) or 3 (to abort).
AdjustTimeStep
This tasks allows the time step to change if the wave speed in any material changes (and the material class supports calculation of wave speed as a function of particle state). Its parameters are:
  • adjustTime (time) - enter the time interval for checking the time step and adjusting it if needed (in alt time units). This parameter is optional; if it is omitted, the time step adjustment is done each time particle data are archived.
  • verbose (value) - if its integer value is not zero, it will print the new time step whenever it changes by more than 1%. If it is zero, the new time step is silently changed. The default it 0.
