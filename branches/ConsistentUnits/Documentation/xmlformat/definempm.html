Defining the Material Points

The material points in MPM calculations are placed within the defined grid. Detailed help on mesh creation is given in the OSUPDocs wiki. In brief, there are two ways to specify the material points. One way is to explicitly list each material point. The other uses one or more "Region" commands that generate the material points. Both these methods must by within a single <MaterialPoints> element, but you are free to use both methods in a single input file. The outline is 

<MaterialPoints>
   <PointList>
     (one <mp> command for each explicit material point)
   </PointList>
   
   (one or more "Region" command)
  
</MaterialPoints>

The subordinate elements in this structure can explicitly list material points (<PointList>) or can define regions to contain points.

Explicit Material Point Command

In a <PointList> block, each material point is defined with

<mp elem='45' mat='1' angle='0' thick='1' conc='0' temp='0'>
  <pt x='1.25' y='-11.25'/>
  <vel x='0' y='0'/>
  <mass m='1e-3'/>
</mp>

The attributes give the element number that contains that point (elem - this attribute is optional; if not provided the element will be looked up), the material number (mat and can be replaced by names using a matname attribute), the material angle (angle in degrees), the thickness (thick in length units, planar 2D only), the initial concentration (conc set as potential from 0 to 1 or you use wtconc to enter the actual weight fraction), and the initial temperature (temp in K).

The elements subordinate to <mp> define the material point location (<pt> and its (x, y, (z if 3D)) coordinates in attributes (in length units in units set b an optional units attribute) and the material point's initial velocity (<vel> and the velocity in the x, y, and z (if 3D) directions (in velocity units or in units set by an optional units attribute).

Normally the particle mass is determined from background grid dimensions. When running a simulation based on material points from a previous simulation (as explain on the OSUPDocs wiki), a subordinate <mass> element can be used to set the particle mass (in mass units or in units set by an optional units attribute).

"Region" Commands

The region commands define a type of material and the its shape commands define what areas are filled with material points. The options are:

  • <Body mat='1' angle='0' thick='1' vx='1000' vy='0' conc='0' temp='0'>
  (Rotation comands)
  (2D shape commands)
  (3D shapes commands)
 </Body>
    Defines a block for one material type. All material points in the shapes created by subordinate commands will be assigned this same material number (mat, which can be replaced by names using a matname attribute), material angle (angle in degrees), thickness (thick length units for planar 2D), initial velocities in the x, y, and z (if 3D) directions (vx, vy and vz in velocity units), the initial concentration (conc set as potential from 0 to 1 where 1), and the initial temperature (temp in K). Any unspecified property will be zero except thickness will match the grid thickness and temperature will match the stress-free temperature.
  • <Hole>
  (2D shape commands)
  (3D shapes commands)
</Hole>
    Defines empty space within a body. A single <Hole> group can contain any number of the shape commands to define regions that are holes within bodies. Material point locations in those shapes will be marked as being empty space and will not be filled by any subsequent regions.
  • <BMP name='GrayScaleFile.bmp' units='mm' width='5.64' height='4.51' angles='GrayAngles.bmp'>
  <Origin x='10' y='20' z='0'/>
  <Intensity mat='1' imin='0' imax='120'>
    <vel x='100' y='200' z='0'/>
    <Thickness >10</Thickness>
    <Angle>0</Angle>
    <Temperature>25</Temperature>
    <Concentration>0</Concentration>
  </Intensity>
  <Intensity imin='0' imax='255' minAngle='0' maxAngle='90'/>
  (Rotation comands)
</BMP>
    This block reads the bit map file (which must be gray scale, non-compressed BMP with 2 to 256 gray levels) specified in the name (as full or relative path name) and assigns material points to the grid based on intensities within the bit map (see OSUPDocs wiki for for details). The width and height attributes (in length units or in units set by an optional units attribute) define the size of the image. You can specify both one or both dimensions; if only one dimension is given, the other will be calculated based on the image aspect ratio. In addition, if the number is negative it is interpreted as the specified length unit per pixel and the total dimension is calculated from number of pixels in the image.
        The subordinate <Origin> maps the origin of the image (lower-left corner in length units or in units set by an optional units attribute) to the grid. In 3D calculations, images are mapped in the x-y plane. You can combine a stack of images in each at different z locations to build up a 3D image. If you add a flipped attribute equal to "yes" to an <Origin> command, the origin will move to the upper-left corner of the image with y increasing in the downward direction and x increasing to the right. This change will flip the image in the y direction in the MPM analysis compared to the image graphics.
        Material points will be assigned to various material types as determined by one or more subodinate <Intensity> blocks. The atttibutes on each <Intensity> block define the material number (mat, which can be replaced by names using a matname attribute) and the minimum and maximum intensity for that material (for 0 to 255).
        The subordinate <Thickness> (planar 2D only), <Angle>, <Temperature>, <vel>, and <Concentration> commands let you set the thickness (in length units or in units set by an optional units attribute), angle (in degrees), temperature (in K), velocity (in velocity units or in units set by an optional units attribute), and concentration potential for the materal points.
        The BMP command can have an optional angles attribute that specifies a second gray-scale file in which the shades of gray indicate the orientation angle for material points. To map the angles gray scale to angle, there should be one <Intensity> command with imin, imax, minAngle, and maxAngle attributes. This command defines a linear interpolation of gray scale intensity (0 for black to 255 for white) to material point angle through the points (imin, minAngle) and (imax, maxAngle).
        Finally, the BMP block can have optional rotation commands to rotate the material axes for all particles defined by the BMP block

Rotation Commands

The rotation commands let you rotate material axes when using anisotropic materials. The commands are:

  • <RotateZ>20</RotateZ>,
<RotateY>y-z</RotateY>,
<RotateX>45</RotateX>
    These commands are to rotate the material axes in the current <Body> to orient anistropic materials in their initial configuration. You apply rotations in order by using one to three commands (although only a single z axis rotation is allowed in 2D simulations). There are two physical pictures. First, you can imagine sitting on the material coordinate system and defining a series of clockwise rotations about material axes that transform the x-y-z analysis axes into the material axes. Alternatively (and identically) the transformation can be viewed as counter-clockwise rotations about analysis axes that transform the material axes into the analysis axes. The text for each command is either an angle or a user-defined function that evaluates to an angle (either are in degrees).
  • <Unrotate/>
    This command cancels the rotations in the current <Body> section. Each shape command in the body use the current rotation transformation. Interspersing this command with new rotations allows each shape to have its own transformations.

2D Shapes

The following commands define 2D shapes (see OSUPDocs wiki for more detail):

  • <Oval units='mm' xmin='10' xmax='40' ymin='10' ymax='40'/>
    An oval bounded by xmin, xmax, ymin, and ymax (in length units or in units set by and units attribute). The parameters can be entered with numeric values or relative to the mesh edges.
  • <Rect xmin='20' xmax='30' ymin='20' ymax='30'/>
    A rectangle bounded by xmin, xmax, ymin, and ymax (in length units or in units set by and units attribute). The parameters can be entered with numeric values or relative to the mesh edges.
  • <Polygon> 
  <pt x='60' y='10'/>
       ⋮
  </Polygon>
    A polygon drawn by the points specified in subordinate <pt> commands (point coordinates in x and y attributes (in length units or in units set by and units attribute). The last point is automatically connected to the first point to make a closed region.

3D Shapes

The following commands define 2D shapes (see OSUPDocs wiki for more details):

  • <Sphere xmin='(xmin)' xmax='(xmax)' ymin='(ymin)' ymax='(ymax)'
                      zmin='(zmin)' zmax='(zmax)'/>
    A spheroid bounded by the parameters xmin, xmax, ymin, ymax, zmin, and zmax (in length units or in units set by and units attribute). The parameters can also be enetered relative to the mesh edges.
  • <Box xmin='(xmin)' xmax='(xmax)' ymin='(ymin)' ymax='(ymax)'
                   zmin='(zmin)' zmax='(zmax)'/>
    A box bounded by the parameters xmin, xmax, ymin, ymax, zmin, and zmax (in length units or in units set by and units attribute). The parameters can also be enetered relative to the mesh edges.
  • Cylinder xmin='(xmin)' xmax='(xmax)' ymin='(ymin)' ymax='(ymax)'
          zmin='(zmin)' zmax='(zmax)' axis='(axis)' radius='(radius)'/>
    A cylinder or cone bounded by the parameters xmin, xmax, ymin, ymax, zmin, and zmax (in length units or in units set by and units attribute). The axis attribute specifies the axis of the cylinder in the 1 (for x), 2 (for y) or 3 (for z) direction. The parameters (except axis) can be entered relative to the mesh edges.
        To create a cone, add a radius attribute with a value between -1 and 1. This value gives the relative radius of the cylinder at the top (if > 0) or at the bottom of the cylinder (if < 0) while the opposite end will have relative radius of 1.
  • <Torus xmin='(xmin)' xmax='(xmax)' ymin='(ymin)' ymax='(ymax)'
       zmin='(zmin)' zmax='(zmax)' axis='(axis)' radius='(radius)'/>
    A torus bounded by the parameters xmin, xmax, ymin, ymax, zmin, and zmax (in length units or in units set by and units attribute). The axis attribute specifies the normal direction to the plane of the torus as in the 1 (for x), 2 (for y) or 3 (for z) direction (default is 3 if not specified). The bounds for the directions in the plane of the torus (e.g., x and y bounds when axis is 3) define an ellipsoid (or circle if the bounds have the same separation) that runs through the center of the torodial ring. The bounds in the direction of the axis define top and bottom for the cross-section of the torodial ring. The cross section will be a circle with radius given by half the distance between the top and bottom. To have an eliptical cross section, you can specify a radius attribute to define the radius of the cross section in the plane of the torus (while top and bottom give radius of the cross section in the axis direction).
  • <Shell axis='(axis)' xmin='(xmin)' xmax='(xmax)' ymin='(ymin)' ymax='(ymax)'
       zmin='(zmin)' zmax='(zmax)' radius='(radius)' thickness='(thickness)'/>
    A cylindrical shell with variable radius and thickness. The (axis) parameter gives the axial direction of the shell. Using the min and max parameters along that axis direction to give the bounds for the shell. You can optionally use the min values for the other two axes to define the origin for the rotation (the default origin is (0,0)). Finally (radius) and (thickness) are user-defined functions of height ("h") along the axis direction to give shell inner radius and thicknes, respectively. All dimensions (including for "h" in the functions) are in length units or in units set by and units attribute.
  • <Polyhedron>
   <faces style='(style)'>
 1.496206e+002  2.879507e+000  1.775602e+002
 5.361129e+001  3.039185e+000  1.498799e+002
 1.496158e+002  2.209645e+000  1.775729e+002
     ...
   </faces>
<Polyhedron>
    Creates arbitrary 3D shape specified by describing the object as a collection of triangular faces. The faces are specified with one or more faces commands subordinate to the Polyhedron command. The current (styles) are:
    • pyramid - define a pyramid shape for which the first point is the apex of the pyramid and the remaining 3 or 4 points define the base. With 3 base points (4 total points), the shape is a tetrahedron; with 4 base points (5 total points), the shape is a pyramid with a quadrilaterial base.
    • tripts or trivectors: define a triclinic shape, which is an eight-corned box for which opposite faces are parallel, but the corners need not meet at right angles. For this style, you specify 4 points (or 12 coordinates). For tripts, the 4 points are the points labeled in the figure below. For trivectors, the first point is point 1 (the origin) and the remaining three points are the vectors between the shape's origin that point.
    • style='15482637': define an arbitrarily shaped 8-corner box. The provided numbers map the order of the provided 8 points (or 24 coordinates) to the standard order shown in the figure below.
    Between the <faces> and the </faces> commands, all the data needed for the selected style are entered in a free format (i.e., numbers delimited by white space, commas, colons, and/or semicolons). The number of points entered must match the number needed for the choosen style. All points are in length units.
