XML Input Files

Deprecated documentation: the new version is in the OSUPDocs wiki.

Finite element calculations using NairnFEA or Material point method simulations using NairnMPM are controlled by input from an XML input file. This input file can be created manually by using any text or XML editing software or created automatically by interpreting commands in a command file using other software such as NairnFEAMPM. NairnFEAMPM is a custom application for Macintosh computers only, but the code engines can run on any computer system.

This section lists the XML tags that are allowed in the input files for NairnFEA or NairnMPM calculations. All tags are case sensitive. Document Type Definition (or DTD) files for NairnFEA or NairnMPM XML files (NairnFEA.dtd and NairnMPM.dtd) are provided in a checkout or in a download of the source code. They are in the /input folder. See File Start section below to learn how to have input files automically validated by a DTD file prior to running the calculations. For better results, all runs should use DTD validation.

Sections of NairnFEA XML Input Files

The following are all the possible sections for input XML files for running NairnFEA. When input files are validated using the DTD file (as recommended), the sections must appear in the specified order. Optional sections that are not needed may be omitted. A sample NairnFEA input file shows a sample calculation of a cantilever beam.

  1. File Start
  2. Header
  3. Defining the FEA Mesh
    1. Explicit Mesh Definition
    2. Generated Mesh
  4. Defining Materials
  5. Boundary Conditions
  6. Temperature (optional)
  7. End the File

Sections of NairnMPM XML Input Files

The following are all the possible sections for input XML files for running NairnMPM. When input files are validated using the DTD file (as recommended), the sections must appear in the specified order. Optional sections that are not needed may be omitted. A sample NairnMPM input file shows a sample calculation of two colliding disks.

  1. File Start
  2. Header
  3. MPM Header
  4. Defining the MPM Background Grid
    1. Generated MPM Grid
    2. Explicit Mesh Definition
  5. Defining the Material Points
  6. Defining Cracks (optional)
  7. Defining Materials
  8. Grid Boundary Conditions (optional)
    1. Generated Grid Boundary Conditions
    2. Generated Velocity Conditions
    3. Generated Temperature Conditions
    4. Generated Concentration Conditions
    5. Explicit Velocity Conditions
  9. Particle Boundary Conditions (optional)
    1. Generated Particle Boundary Conditions
    2. Generated Particle Load Conditions
    3. Generated Heat Flux Conditions
    4. Generated Concentration Flux Conditions
    5. Explicit Particle Loading
  10. Thermal Conduction and Loading (optional)
  11. Adding Gravity Field (optional)
  12. Custom Tasks (optional)
  13. End the File
