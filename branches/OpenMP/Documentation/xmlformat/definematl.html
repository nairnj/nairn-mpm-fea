Defining Materials

NairnFEA and NairnMPM support a variety of material types. An analysis can mix any number of materials. The materials are defined in a series of Material tags. A specific material is defined with:

<Material Type='1' Name='Polymer'>
     .
     .
     .
</Material>

where Type is the material type (integer) which defines the type of material and Name is any user-defined description of the the material (<200 characters and most be unique among defined materials if you reference them by name). The material properties for each type are defined by subordinate tags within the Material element.

When materials are assigned to FEA mesh elements, MPM materials points, MPM crack tips, or in a few other commands, they can be referenced by number or by name:

By Numbers
In this method, you use mat="#" attributes where # is the material number. The materials are defined by numbers in the order they appear in the output file with the first material being number 1. You have to be careful to use the correct number. If you add new materials to a file, it is best to add them to the end of the materials list, otherwise previous commands that referenced materials after an inserted material, will point to the wrong material.
By Name
In this method, you use matname="A Name" attributes where "A Name" is a unique material name that matches the name of any defined material in the file. You can use these names even before the materials are defined, but errors will occur if you reference materials that are not eventually defined or if the defined materials do not have unique names.

All defined materials will appear in the output file in the ordered referenced by name rather than in the order defined in the input file. But, if you do not reference any materials by name, they will appear in the order found in the input file. If some, but not all, materials are referenced by name, those that were not referenced will be output at the end of the materials in the output file by their order in the input file. If you mix material attributes by number and by name, the numbers uses have to refer to how they appear in the output file and not in the input file. For this reason, it is hard to mix the two methods and that approach should be avoided. Finally, if you use both mat and matname attributes in a single command, the matname one will be used and the mat will be ignored.

Certain materials work in NairnFEA and NairnMPM, while others are limited to NairnFEA or NairnMPM. Similarly, not all materials for NairnMPM can be used in 3D calculations. The 2D NairnMPM columns has two marks. The first means it works for planar MPM while the second indicates it works for axisymmetrix MPM. Material Properties for all Material Types

There are some material properties that are common to all material types. These properties are only used in NairnMPM and are not allowed in NairnFEA. Some of these properties only work for selected materials (refer to each material for possible details). The common properties are

<rho>1.5</rho>
The material density (in g/cm3) (can be used in NairnFEA files, but it is ignored).
<Cp>1.5</Cp> or <Cv>1.5</Cv>
The material's heat capacity (in J/(kg-K)). It is used when doing conduction calculations and/or coupled mechanical energy and by some material constitutitive laws. Both Cp and Cv set the same (and only) heat capacity used in the MPM calculations. MPM is a strain-drive method, which implies the heat capacity should be interpreted as the constant-volume heat capacity.
<csat>0.5</csat>
The material's saturation concentration entered as a weight fraction. Only used when doing diffusion calculations.
<Propagate criterion='1' direction='0' traction='1'/>
<AltPropagate criterion='7' direction='4' traction='1'/>
These properties let each material have a different primary and secondary propagation criterion and/or directions than the ones defined in the MPM Header. Set these criterion as described for the Propagate and AltPropagate commands in the MPM Header. To activate propagation:
  1. The MPM Header must have a Propagate command.
  2. The MPM Header can optionally have an AltPropagate command.
  3. Any material can override the default primary and/or secondary criteria by using a Propagate and/or AltPropagate command within its material defintion. Even if the MPM Header does not have an AltPropagate command, a material can add a secondary criterion with its own AltPropagate command.
<KIc>2.3</KIc>
Critical mode I stress intensity factor (in MPa m1/2). Only used when there is crack propagation with criteria 1, 4, or 5.
<KIexp>2.3</KIexp>
Exponent p in the elliptical criteria for crack growth. Only used for crack propagation with criterion 5.
<KIIc>1.2</KIIc>
Critical mode II stress intensity factor (in MPa m1.2). Only used when there is crack propagation with criterion 5.
<KIIexp>2.3</KIIexp>
Exponent q in the elliptical criteria for crack growth. Only used for crack propagation with criterion 5.
<JIc>2.3</JIc>
Critical mode I energy release rate (in J/m2). Only used when there is crack propagation with criteria 2, 3, or 7. For criterion 2, it is only used if the initTime is not being used. Also used by traction-law materials.
<JIIc>1.2</JIIc>
Critical mode II energy release rate (in J/m2). Currently only used by traction-law materials.
<nmix>2</nmix>
A parameter used in some mixed-mode failure laws (e.g., to implement failure when (GI/JIc)nmix + (GII/JIIc)nmix = 1 in TriangularTraction cohesive law).
<initTime>3</initTime>
Time (in ms) to start crack growht. Only used when there is crack propagation with criteria 2. For criterion 2, initTime takes precidence of a JIc property.
<speed>5</speed>
Define the constant crack speed (in m/sec). This speed, however, is only active when there is crack propagation with criterion 2.
<maxLength>20</maxLength>
Define the maximum length of the crack (in mm). The crack growth will stop when this length is reached. This limit, however, is only active when there is crack propagation with criterion 2.
<gamma>.5</gamma>
Only used when there is crack propagation with criterion 3. Still in development and not meant for general use.
<p>.5</p>
Only used when there is crack propagation with criterion 3. Still in development and not meant for general use.
<gain>10e5</gain>
Only used when there is crack propagation with criterion 3. Still in development and not meant for general use.
<xGrow>20</xGrow> and <yGrow>0</yGrow>
These properties (one or both with default 0) specify a unit vector for a constant crack growth direction. They only apply for crack propagation with criterion 2 and then only when that criterion selects the default propagation direction. The result is constant crack growth direction regardless of stress state or crack tip orientation. Any input vector will be normalized to a unit vector. If constant crack growth directions with a fixed crack are located precisely on grid lines, it is possible for the crack algorithm to not recognize the crack plane. It is better to move such a crack sligthly off grid lines.
<delIc>0.1</delIc> and <delIIc>0.1</delIIc>
Critical crack opening displacement. Only used when there is crack propagation by criterion 6. Also used by traction-law materials.
<constantTip>0</constantTip>
Set to 0 or 1 where 1 means the crack tip will always uses that material even if the crack is in or moves to another material. This property is useful to have multiple cracks in the same material with different fracture properties by defining multiple materials that are identical except for fracture properties, defining problem with one of those materials, but use other materials for each crack tip's material. Default is 0.
<ArtificialVisc/>, <avA1>.2</avA1>, and <avA2>2.0</avA2>
This first empty tag turns artificial visocsity on, which adds a pressure, Q, related to velocity gradient on the particle, but only when it is compressing. The equation is
Q = Δx*|Dkk|*(A1*C + A2*Δx*|Dkk|)
where Δx is the cell size of the mesh, |Dkk| is the relative volume change rate, C is the bulk wave speed in the material, and A1 and A2 are adjustable constants, which are entered using the above <avA1> and <avA2> commands. The defaults are off, A1=0.2, and A2=2.0. It is common, but not required, to choose A2 = 10*A1.
    The artificial viscosity property is supported in some materials (as documented under each material). If you use these commands in a material that does not support, an error will result.
<Friction mat='2'>0.3</Friction>
This command within a material definition allows multimaterial mode to have different contact laws between different pairs of materials. Set the contact law as explained for Friction command in the Cracks section of the MPM header except that a setting less than -10 means to assume materials are always in contact and revert to single material mode. The mat attribute defines contact mechanics between this material and the one listed (self contact is ignored). Alternatively, the other material can be specified by name using a matname attribute as explained here. If a node has more than two materials, the contact law used will be one between each material and the other material with the most volume.
    You only need to set friction on one of the materials in a pair. If you set both, the setting for the second material will override the one from the first material.
<color red='0.1' green='0.5' blue='0.3' alpha='1.0/>
Sets the color of the material with red, green, blue, and alpha values between 0.0 and 1.0. The color may be used by post-analysis software, but it has no affect on the calculations.

Imperfect Interface Material

The material type is Type=5 and is used to model imperfect interfaces. The input properties are

<Dn>200</Dn>
Stiffness for loading the interface in the normal direction (in MPa/mm)
<Dt>20</Dt>
Stiffness for loading the interface in the tangential direction (in MPa/mm)

Imperfect interfaces allow displacement discontinuties along the interface. The discontinuity in the normal direction is σn/Dn and in the tangential direction is τt/Dn where σn is the normal component of the traction on the interface and τt is the tangential component. Interface parameters of Dn = Dt = ∞ leads to zero displacement discontinuity or a perfect interface; Dn = Dt = 0 leads to zero stress or a debonded interface. All other finite values define an imperfect interface. The interface is linear and thus compressive normal stress will lead to non-physical negative displacement discontinuties or interface overlap. If an interface is in compression, then Dn should be infinite, at least over regions of compressive stress.

Mooney-Rivlin Hyperelastic, Isotropic

The material type is Type=8. It is a large-deformation, isotropic, hyperelastic material. The stresses are defined from the strain energy, which is defined by invariants of the left Cauchy Green Tensor ([B] = [F][F]T where [F] is the deformation gradient). In a Mooney-Rivlin material, the strain energy depends on three material properties:

W = (1/2)G1(I1-3) + (1/2)G2(I2-3) + U(J)

where I1 = Trace([B])/J2/3, I2 = 0.5(I12 - [B]..[B]/J4/3), and J = Det([F]). The remaining parameters are material properties:

<G1>0.1</G1>
The G1 shear modulus (in MPa). The low-strain shear modulus is G1+G2.
<G2>0.33</G2>
The G2 shear modulus (in MPa). The low-strain shear modulus is G1+G2.
<UJOption>0</UJOption>
This selects the dilational energy term U(J) and the possible integer values mean:
    0: U(J) = (K/2)((1/2)(J2-1) - ln J)
    1: U(J) = (K/2)(J-1)2
    2: U(J) = (K/2)(ln J)2
where K is the bulk modulus and is entered as a separate property. The default (and recommended) option is 0.
<IdealRubber/>
Including this command makes the material an ideal rubber elastic material. An ideal rubber means that during isothermal loading, there is no change in internal energy. In other words, all the work is turned into heat. In adiabatic loading, this heat causes the temperature to rise.
<K>2.5</K>
The K modulus (in MPa), which is equal to the low-strain bulk modulus.
<alpha>60</alpha>
The material thermal expansion coefficient (in ppm/C)
<beta>60</beta>
The material concentration expansion coefficient (in strain/wt fraction)
<D>400</D>
Diffusion constant (in mm2/sec). Only used when doing diffusion calculations.
<kCond>400</kCond>
Thermal conductivity (in W/(m-K)). Only used when doing conduction calculations. NairnMPM only.
Other Properties
See the properties common to all materials. This material supports artificial viscosity.

A neo-Hookean material is a special case of a Mooney-Rivlin material by setting G2 = 0.

History Data for Mooney-Rivlin Materials

This material use history #1 to store the volumetric strain (i.e., the determinant of the deformation gradient). The total strain, which is elastic, is stored in the elastic strain variable, but the plastic strain stores the left Cauchy Green tensor.

This material tracks entropy in the plastic energy for the particle. The entropy is net effect of heat flow by external heating, by conduction, or when set to be an ideal rubber, by internal mechanisms. For example, when an ideal rubber is loaded isothermally in tension, the force is related to entropy by

F = -T (dS/dL)T

An ideal rubber in NairnMPM correctly follows this law.

History Data for Isotropic Hyperelastic-Plastic Materials

The first history variables are determined by the hardening law. After those variables, the next history variable stores the volumetric strain (i.e., the determinant of the deformation gradient, [F]). The total strain, which is elastic, is stored in the elastic strain variable, while the plastic strain stores the elastic, left Cauchy Green tensor.

Ideal Gas, as a Hyperelastic Material

The material type is an ideal gas, implemented as a large-deformation, isotropic, hyperelastic material at finite deformations. Its contitutive law is.

P = J-1 P0 T/T0

where J is determinant of the deformation tensor and J = V/V0. The pressure P is stored in the normal stresses or σxx = σyy = σzz = -P. The material can be used in 3D and 2D plain strain calculations, but not in 2D plane stress calculations.

The possible input properties are:

<P0>0.1013</P0>
The reference pressure (in MPa) at reference temperature and reference density. This must be a positive value greather than zero.
<T0>273.15</T0>
The reference temperature (in Kelvin)
<rho>0.00129</rho>
The reference density (in g/cm3) at reference temperature. This must be a positive value greather than zero.
<Cv>1</Cv>
If heat capacity is omitted or is less than or equal 1, CV is set to (3/2)R for a monotonic gas. If the enter value is greater than 1, CV is set to (5/2)R for a diatomic gas.
Other Properties (where applicable)
See the properties common to all materials.

When using gas particles, the stress free temperature must be set to the desired temperature in Kelvin. The material properties refer to gas state at any reference conditions. For example, air might have reference pressure of 1 atm = 0.101325 MPa at any temperature T0. The density of air (or any gas) is then given by

rho = (P0/T0) * (Mg/R)

where Mg is the molecular weight of the gas (such as 28.97 g/mol for air) and R is the gas constant, R = 8.3144621 J/k/mol.

This material stores total work energy (-P dV work) in the particle's strain energy and change in entropy in the particle's plastic energy.

When EnergyCoupling is off, this material models isothermal compression and expansion, which implies all work results in heating or cooling. The amount of heat generated is tracked in the particle's heat energy. The problem may include heat input (in thermal coundary conditions), which may cause temperture rises. In other words, this mode only means the gas itself will not cause temperature changes. The internal energy will not change unless there is external heating and the plastic energy will give total work.

To model adiabatic compression and expansion, activate EnergyCoupling. This mode will convert work to heat resulting in heating during compression or cooling during expansion. You need to enter heat capacity (which can only pick monotonic or diatomic gas) and thermal conductivity. The current implementation uses a temperature independent conductivity, which may change in the future.

Ideal gas particles are fairly stable, but can be made unstable by certain boundary conditions on constraining walls. If stability problems arise, try different boundary conditions. They also do not work for irreversibale processes such as free expansion into empty space. They are intended to be always bounded by stable pressure.

The transition between the two states is determined by a dilation rule, a distortion rule, or a Von Mises stress rule (see transition property). When using a dilation rule, the new stress-strain relation can include a changed offset in volumetric strain corresponding to a stress-free conditions at a non-zero dilation relative to the initial state. This change normally leads to an instantaneous change in stress upon transition. When using a distortion or Von Mises stress rule, the offset is ignored and the change is only a change in slope of mechanical properties. The properties determining the transition are:

<critical>0.02</transition>

<reversible/>
Make the transition reversible (the default is reversible).
<irreversible/>
Make the transition irreversible (the default is reversible).

Rigid Material

The material type is Type=11 and it is actually two materials in one, depending on the value for its SetDirection property:

  1. If SetDirection is 0 through 7, the material is not really a material and points that are this type of rigid material will not really be part of the MPM analysis. These rigid material points are used to set boundary conditions on the grid that move with the particles. The rigid material points should be initialized to a velocity, temperature, or concentration when they are created. As the rigid material points move through the grid, all nodes connected to those points will be assigned grid-based boundary conditions for the selected particle properties of velocity, temperature, and/or concentration. To have the boundary condition vary in time, use the the SettingFunction(2)(3) properties below to select the value rather than setting the initial particle properties (which can only be constant).
  2. If SetDirection is 8, the material will correspond to actual material points that move through the analysis. Since they are rigid, they will move with their prescribed velocity. They will interact with other material points only through multimaterial contact laws and thus this type of rigid material can only be used in multimaterial mode MPM calculations. These particles will also ignore grid-based velocity boundary conditions, including those set by rigid particles with SetDirection equal to 0 through 7. Rigid contact is briefly described in Lemiale et al. (2010).
        The advantage of this type of rigid material (unlike the other type) is that you can set frictional properties for contact between rigid materials and non-rigid materials. These rigid materials will move in the direction defined by the initial velocity. To have non-constant (e.g., rotational) motion, use the SettingFunction(2)(3) properties below rather than the initial partical velocity. If two different rigid materials of this type interact at the same node, however, there is a conflict and the calculation will stop. Finally, this type of rigid material can only set velocity; it cannot set temperature or concentration.
        This type of material ignores cracks, but that actually makes to possible to use them for new crack problems. For example, these rigid particles can provide wedge opening of a crack. If the rigid particles run along an explicit crack plane, those particles may contact particles on either side of the crack and then move the top and bottom surfaces of the crack apart. It may be the only way to wedge open a crack (i.e., the only material that can act like it is within a crack rather than on either side of the crack). All crack calculations (e.g., crack contact, traction laws, imperfect interfaces, updating of crack position, and J integral) will ignore the rigid particles. The interactions with the rigid particles arise only by contact laws between the rigid particles and the various crack velocity fields.

The rigid material properties determine which boundary conditions are set:

<SetDirection>1</SetDirection>
Values 1 through 7 set various combinations of velocity directions. To create the alternate type of rigid particles that can implement frictional contact, set this value to 8. The options 1 through 7 set grid velocites to various components of the rigid particle velocity by bit-wise selection with first three bits being for x, y, and z directions (or R, Z, θ if axisymmetric):

<SetTemperature/>
Set temperature to the particle temperature. This setting has no affect unless the analysis is doing thermal calculations. Can only be used if SetDirection is 0 through 7. Use 0 to having moving heat source only; use 1 to 7 to combine moving heat source with velocity boundary conditions.
<SetConcentration/>
Set concentration to the particle concentration entered as a potential from 0 to 1 where 1 is the saturation concentration for the particle's material type. This setting has no affect unless the analysis is doing diffusion calculations. Can only be used if SetDirection is 0 through 7. Use 0 to having moving concentration source only; use 1 to 7 to combine moving concentration source with velocity boundary conditions.
<SettingFunction>50*(1-sign(t-10))</SettingFunction>
<SettingFunction2>50*(1-sign(t-10))</SettingFunction2>
<SettingFunction3>50*(1-sign(t-10))</SettingFunction3>
By entering user-defined functions of time and position the rigid particle velocity setting can vary with time and position. The functions should evaluate to the velocity in mm/s. The initial velocity assigned to the rigid particles when they are created will be ignored.
     The three setting functions can set up to three directions. For SetDirection 1 to 7 the functions must be given in order (SettingFunction, SettingFunction2, and SettingFunction3) and apply to the set velocities in order (e.g., when controlling y and z with SetDirection=6, the SettingFunction is for y direction and SettingFunction2 is for z direction). For SetDirection of 8, the three functions are for setting x, y, and z directions and zero to three can be used. Any not controlled will use the initial particle velocity.
<ValueFunction>50*(1-sign(t-10))</ValueFunction>
By entering user-defined functions of time (t in ms) and position (x, y, and z in mm) the rigid particle temperature and concentration can vary with time and position. The function should evaluate to the temperature or concentration potential. The initial temperature or concentration assigned to the rigid particles when they are created will be ignored.
     Since there is only one value function, you cannot independently vary temperature and concentration with time and position. Thus rigid particles with value functions should only set one of these quantities.

Note that multimaterial mode MPM calculations can combine moving boundary condition rigid particles (SetDirection=0 to 7) with contact rigid particles (SetDirection=8) to provide different functions, but single-material mode MPM can only use boundary condition rigid particles (SetDirection=0 to 7).

Note that creating rigid particles but not setting any velocity direction (SetDirection=0) can be used to create moving temperature or concentration boundary conditions that move over the particles; i.e., the rigid particles will move through the real particles. Such moving temperature or concentration boundary conditions can only move at a constant velocity defined by the initial particle velocity.

If a rigid particle is added to a problem, but it does not set any velocity, temperature, or concentration (SetDirection=0 or omitted and SetTemperature and SetConcentration omitted), it will have no affect on the calculations. It may plot in visualization software and therefore appear to be part of the simulation, but it will be identical to the results that would be obtained if it had never been added to the problem.

Hill Plastic, Orthotropic Material

The material type is Type=15. This material is a small-strain, elastic-plastic material based on an orthotropic material. The material is elastic plastic, but has an anisotropic crition to allow yielding at different stresses when loaded in different directions. Enter the underlying orthotropic material properties and then enter the direction-dependent yield stresses. Any omitted yield stress will be assumed to be inifinite (i.e., no yielding in that direction). At least one yield stress is required (otherwise just use an orthotropic material):

Main Properties
See Linear Elastic, Orthotropic material for most material properties.
<yldxx>10</yldxx>
Yield stress from uniaxial tension test the material's x direction. Enter in units of MPa.
<yldyy>10</yldyy>
Yield stress from uniaxial tension test the material's y direction. Enter in units of MPa.
<yldzz>10</yldzz>
Yield stress of uniaxial tension test the material's z direction. Enter in units of MPa.
<yldxy>10</yldxy>
Yield stress from pure shear test in the material's x-y plane. Enter in units of MPa.
<yldyz>10</yldyz>
Yield stress from pure shear test in the material's y-z plane. Enter in units of MPa.
<yldxz>10</yldxz>
Yield stress from pure shear test in the material's x-z plane. Enter in units of MPa.
<khard>20</khard>
Hardening term (see below)
<nhard>20</nhard>
Exponent on cumulative plastic strain in hardening term (see below)

The anisotropic yield criterion is

[ F(σyy - σzz)2 + G(σxx - σzz)2 + H(σyy - σxx)2 + 2Lτyz2 + 2Mτxz2 + 2Nτxy2 ]1/2 - (1 + Kεpn)

where σxx, σyy, σzz, τyz, τxz, and τxy are stresses in the material axis system after rotation from the anaysis coordinates. The constant are determined by the yield stresses:

2F = (1/yldyy2) + (1/yldzz2) - (1/yldxx2)
2G = (1/yldxx2) + (1/yldzz2) - (1/yldyy2)
2H = (1/yldxx2) + (1/yldyy2) - (1/yldzz2)
2L = (1/yldyz2)
2M = (1/yldxz2)
2N = (1/yldxy2)

The τyz and τxz terms and their yield stresses are not needed for 2D analyses.

The yield stress have to be selected such that the plastic potential is positive semidefinite. Analysis shows that all tensile yield stress must satisfy:

( (1/yldii2) - (1/yldjj2) )2 ≤ 1/yldkk2 ≤ ( (1/yldii2) + (1/yldjj2) )2

In more practical terms, if two yield stresses are related by some ratio R = yldii/yldjj, then the third yield stress is bracketed by:

yldii/|1-R| ≤ yldkk ≤ yldii/(1+R)

For R=0, yldkk=yldii. As R→∞, yldkk=yldjj.

The hardening terms, K and n, are dimensionless; εp is cumulative plastic strain.

History Data for Von Mises Plastic, Isotropic

The history variable is the cummulative equivalent plastic strain defined as the sum of sqrt(2/3)||dεp||. This variable can be archived as history variable 1.

These two material types use Type=17 and Type=25. Both are isotropic materials with plasticity and find pressure using the Mie-Grüneisen equation of state by large deformation methods. The Type=17 material uses low-strain, hypoelastic methods for shear and yielding. The Type=25 material uses large-deformation for shear and yielding. Both materials can use any hardening law. The material properties for both these materials are:

<C0>4004</C0>
The bulk wave speed entered in m/sec.
<S1>1.35</S1>
Parameter in Mie-Grüneisen equation of state. It is dimensionless.
<S2>0</S2>
Parameter in Mie-Grüneisen equation of state. It is dimensionless.
<S3>0</S2>
Parameter in Mie-Grüneisen equation of state. It is dimensionless.
<gamma0>1.64</gamma0>
Parameter in Mie-Grüneisen equation of state. It is dimensionless.
<G>160000</G>
Shear modulus, G0, at zero pressure and the reference temperature. Enter in units MPa.
<Hardening>Linear</Hardening>
This command selects the hardening law. It should be followed by entering yielding properties required by the hardening law.
Other Properties
Enter density, ρ, and heat capacity, CV, in properties common to all materials. This material supports artificial viscosity.

In this material, the pressure is related to the volumetric compression strain (x = -ΔV/V0) by the Mie-Grüneisen equation of state. When x>0, the material is in compression and pressure is

P = ρ0C02x(1 - 0.5 γ0 x) / (1 - S1x - S2x2 - S3x3)2 + ρ0γ0e

where e is the internal energy, which is sum of work energy (stored in strain energy) and heat energy. The heat energy depends on temperature relative to a reference temperature; the reference temperature is set using the StressFreeTemp command. When x<0, the material is in tension and the pressure changes to

P = ρ0C02x + ρ0γ0e

The first term is bulk modulus terms with the low-strain bulk modulus being ρ0C02. For more details on this equation of state see Wilkens (1999), page 61 (although the final pressure form here is different but is equivalent if compared as polynomial expansions; this form also includes S2 and S3 options while Wilkens only has S = S1).

The hyperelastic version uses shear energy function of

W = (1/2)G(I1-3)

which is the first term is same as for a Mooney-Rivlin material.

This equation of state automatically handles thermal expansion by inclusion of thermal energy in the internal energy. You do not need to enter a thermal expansion coefficient because it is determined by other properties through:

α = (γ0CV)/(3C02)

which means you need to enter heat capacity (note that this is a linear expansion coefficient).

This material allows isotropic conduction and diffusion by entering isotropic transport properties (kCond, D, and beta) and bulk properties (heat capacity and csat).

<sigmaI>10</sigmaI> and <sigmaII>100</sigmaII>
The peak stress or cohesive stress of the traction law (σ). Enter in units of MPa.
<JIc>10</JIc> and <JIIc>100</JIIc>
The toughness of the traction law or area under the traction law (Jc). Enter in units of J/m2.
<delIc>10</delIc> and <delIIc>100</delIIc>
The maximum opening displacement or critical displacement (δc) at which debonding occurs. Enter in units of mm.

The relative displacement at the location of the peak in the triangular law. Enter in dimensionless units as fraction of delIc or delIIc (0 to 1). You can enter zero to get pure linear softening. You must enter zero or one for the next two properties. If one is provided, the other will be calculated from k = σmax/(δpeak δc). A valid traction law requires δpeak ≤ 1. If neither k nor δpeak are entered, the peak will be located at δpeak = 0.225926299. This location provides the closest match to the cubic traction law for area under the curve as a function of δ.

The theoretical J vs δ curve will be flat up to δpeak δc and then will increase linearly up to δc with a slope of

500 σmax/(1-δpeak)

These entered traction laws give mode I or mode II traction law for pure mode I or pure mode II loading. For mixed-mode loading, the normal and and shear tractions still depend only on normal and shear displacements, respectively, but the conditions for failure change. Under pure mode loading, failure occurs with the relevent displacement reaches the critical displacement. For mixed-mode loading, failure occurs when:

(GI/JIc)nmix + (GII/JIIc)nmix = 1

where GI and GII are areas under mode I and mode II traction laws up to current displacements. Now this failure criterion will mean that failure may occur when normal or shear tractions are nonzero and result in a sudden drop to zero traction. This approach is identical to the one adopted by Thouless et. al to handle mixed-mode loading and appears to work well. There has been work to modify the traction laws so they always approach zero at time of failure (e.g., work of Hogberg (2006) or Tvergaard and Hutchinson (1994)). These approaches were tried, gave poor results for certain problems, led to non-physical coupling between mode I and mode II, and therefore were abandoned.

Jtotal=(1/2)(σ1δ2+σ2(δc-δ1))

which can be divided into two processes. The area under the first peak and bounded by the dotted red line in the above figure, is the total energy released by the first process. CoupledSawTooth Traction Law Material

The material type is Type=23. This material is a traction law material for crack surfaces and therefore can only be used on cracks or when creating new crack surfaces by the Propagate option. The traction law is the same as for the triangular law:

except that cod and traction are now effective terms that are related to normal and tangential components of traction and displacement:

Traction = Teff where Teff2 = Tn2 + Tt2
cod = δeff where δeff2 = δn2 + δt2

The use of effective terms is meant to account for mixed mode effects in a single coupled traction law. The properties you enter are as follows:

<sigmaI>10</sigmaI>, <JIc>10</JIc>, <delIc>10</delIc>, <kIe>200</kIe>, <delpkI>0.1</delpkI>
These properties are defined in the Triangular Traction Law and they all mean the same thing for this traction law, except they apply now to effective traction and critical effective cod rather than mode I traction and critical cod. JIc is the total energy release rate from area on the effective traction - effective cod curve.

This law can be expressed using damage mechanics by defining an effective slope of keff = (1-D)k where k is the initial slope of the curve and D is a damage parameter. This effective slope describes a line from the origin to the point on the triangular curve corresponding to the maximum displacement seen on the current particle and the effective traction is Teff = keffδeff. The effective stiffness is found from

(1-D) = δpk*(δc-δmax)/(δmax*(δc-δpk))

where δc is the critical effective cod and δmax is the maximum effective cod seen on the crack particle, except that it is initialized to the peak cod (δpk) at the start of the calculations. By this relation, D = 0 until the effective cod passes the peak cod. Thereafter, D increases from 0 at the peak to 1 at the critical cod. Once D is found, the normal and tangential tractions are Tn = keffδn and Tt = keffδt, which satisfy the effective traction - effective cod law given above. Note that during monotonic loading the traction will remain on the curve. If unloading occurs, however, the tractions follow a line back to the original from the maximum cod.
