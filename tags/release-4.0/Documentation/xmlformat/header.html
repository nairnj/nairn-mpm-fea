Header

This required section defines the type of analysis to be done and allows setting of some optional parameters. 

<Header>
   <Description>
Title: MPM Analysis
Name: User Name
    Sample MPM problem
   </Description>
   <Analysis>11</Analysis>
   <Output>YYYYNY</Output>
   <Select node='1'/>
   <Select node='23'/>
   <DevelFlag Number='0'/>2</DevelFlag/>
</Header>

where

<Description>
Provides an optional free-format, text description of the problem. The description must begin on the line after the tag. All characters up to the ending tag are part of the description.
<Analysis>
Defines type of analysis to perform (required). The data is an integer code with the following options:
  • 0 = Plane strain FEA analysis (NairnFEA only)
  • 1 = Plane stress FEA analysis (NairnFEA only)
  • 2 = Axisymmetric FEA analysis (NairnFEA only)
  • 10 = Plane strain MPM analysis (NairnMPM only)
  • 11 = Plane stress MPM analysis (NairnMPM only)
  • 12 = 3D MPM analysis (NairnMPM only)
  • 13 = Axisymmetric MPM analysis (NairnMPM only)
In axisymmetric calculations, the x axis becomes the radial (or R) axis, the y axis becomes the axial (or Z axis), and the z axis is for hoop stress and strain.
<Output>
This optional tag is for NairnFEA only. It selects the desired output for the finite element results file. It can be used to reduce output file size to desired results only. The text of the tag has 1 to 6 letters, each of which must be 'Y', 'C', or 'N. A 'Y' means that result is fully included in the file. A 'C' means that result is conditionally included only if it is relevant to one of a set of preselected nodes. A 'N' means that result is not included in the file. The six output result options are:
  • Byte 1: Displacement of each node
  • Byte 2: Nodal forces within each element
  • Byte 3: Nodal stresses within each element
  • Byte 4: Average nodal stresses at each node
  • Byte 5: Force reactivities at nodes with fixed displacement
  • Byte 6: Strain energy of each element and total strain energy
Note that if certain results are omitted, they will be not be available for plotting in analysis software. Bytes 2 and 3 are element results. If these options are 'C' they will output the full results for any element that has one or more preselected nodes. Bytes 5 and 6 provide full output for both option 'Y' and 'C' (i.e., there are not sensitive to which nodes are selected). Finally, if no Output tag is used, the default settings are 'YYYYNY'.
<Select node='23'>
These optional tags can be used in conjunction with an Output tag (NairnFEA only) to preselect nodes for any output setting of 'C'. The node attrribute selects which node should be selected. This method of preselecting nodes is typically only used when doing explicit definition of an FEA mesh. When generating a mesh with higher level commands, it is preferrable to select nodes for conditional output using generated boundary condition commands.
<DevelFlag Number='0'/>2</DevelFlag/>
This tag sets a development flag. You can set up to 5 flags by using Number from 0 to 4. The default for all flags is zero. Any non-zero flags will be listed in the output file after the description. These flags are for use in development. The code can be modified to alter the calculation depending on a certain flag. They should not be relied upon to mean anything permanent.
