File Start

All input files must begin with an XML declaration, an optional document type declaration, and the main input file tag. The start for NairnFEA is: 

<?xml version='1.0'?>
<!DOCTYPE JANFEAInput SYSTEM "NairnFEA.dtd"
[
  <!ENTITY length "200">
  <!ENTITY width "100">  
]>
<JANFEAInput version='3'>

The start for NairnMPM is:

<?xml version='1.0'?>
<!DOCTYPE JANFEAInput SYSTEM "NairnMPM.dtd"
[
  <!ENTITY maxtime "1e-03">
  <!ENTITY width "100">  
]>
<JANFEAInput version='3'>

By default, the XML input file will not be validated. If you want to do validation, which is highly recommended, run the code with the validation option (-v) and provide document type information in the optional !DOCTYPE element. When that line is included and -v is used, the input XML file will be validated using the Document Type Definition (or DTD) file which must be at the path name or URL specified after the SYSTEM keyword. The required DTD files are provided in the /input folder of the source code file. You can reference this file or move it to any accessible location. If the DTD file is not found, the code will exit with an error message. If the !DOCTYPE element is not given, validation will be skipped and the code will continue with no error message even if the -v option is used. When validation is not done, the code itself has its own error checks that are still performed, but it always preferrable to provide the DTD file because that step is fast and provides more robust error checking.

When using validation, all sections in the file must appear in the order listed in the file format index. Some of these elements are required, some elements can appear only once while others can appear multiple times, and some elements are optional. If there is a file error or warning, an error message will be written to standard error.

The !DOCTYPE element can define any number of entities. These entities can be used elsewhere in the text of the XML file and they will be replaced by the text in the entity definition. For example, the above MPM file can use the entity &maxtime; and it will be replaced with the text 1e-03. Entities can define values that make it easier to read input XML files and easier to modify them for new calculations.

The 'version' attribute on the JANFEAInput tag gives the version number for the file. It is currently not used and may be omitted.
