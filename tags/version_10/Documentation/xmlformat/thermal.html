Thermal Conduction and Loading and Thermodynamics

NairnMPM can do coupled elasticity-thermal conductivity calculations, provide thermal boundary conditions, convert various material processes into heat, and/or or can apply a constant temperature difference to all particles (as detailed on the OSUPDocs wiki). Thermal stresses are always calculated relative to the stress free temperature, which is set in the MPM header block (it is set to zero by default, but some materials need it to be set to actual, absolute temperature). Other thermal effects are set up with the following block:

<Thermal>
  <Conduction/>
  <CrackTipHeating/>
  <CrackContactHeating/>
  <ContactHeating/>
  <EnergyCoupling/>
  <NoAVHeating/>
  <Isothermal time="5" start="2">-200</Isothermal>
</Thermal>

where

  • <Conduction/>
    This tag specifies that the calculation should do coupled thermal conduction analysis. 
  • <CrackTipHeating/>
    This tag will cause energy released by crack growth to be released as a heat source into a conduction analysis. This command has no affect unless the <Conduction/> is activate and you must be sure to correctly et the crack thickness. 
  • <ContactHeating/>
    This tag will activate conversion of the work of frictional sliding between materials in multimaterial mode into heat.
  • <CrackContactHeating/>
    This tag will activate conversion of the work of frictional sliding between crack surfaces into heat.
  • <EnergyCoupling/>
    This tag will cause energy dissipated by a material's constitutive low to be converted to a heat source that will increase temperature. In other words, the particles will be locally adiabatic. This option can be used with or without conduction. If this option is omitted, the simulation will be locally isothermal. 
  • <Isothermal time="5" start="2">-200</Isothermal>
    This tag specifies the constant temperature difference (vs. stress free temperature) to apply to each particle in degree K. The optional time attribute allows the simulation to ramp the temperature difference from zero to the difference supplied in the <Isothermal> value over any time span (in ms). The optional start attribute allows the ramp to start at any time (in ms).
