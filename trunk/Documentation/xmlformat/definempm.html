Defining the Material Points

The material points in MPM calculations are placed within the defined grid. There are two ways to specify the material points. One way is to explicitly list each material point. The other uses one or more commands that generate the material points. Both these methods must by within a single <MaterialPoints> element, but you are free to use both methods in a single input file.

Material points are defined in a <MaterialPoints> element. The subordinate elements in this structure can explicitly list material points (<PointList>) or can define regions to contain points (<Body>), regions to be free of points (<Hole>), or map points from a gray-scale image (<BMP>). The general format (which shows both 2D and 3D commands, although they cannot be mixed in actual command files) is: 

<MaterialPoints>
   <PointList>
      <mp elem='45' mat='1' angle='0' thick='1' conc='0' temp='0'>
        <pt units='mm' x='1.25' y='-11.25'/>
        <vel units='mm/sec' x='0' y='0'/>
      </mp>
      <mp mat='1' angle='0' thick='1' wtconc='.02'>
        <pt x='3.75' y='-11.25'/>
        <mass units='g' m='1e-3'/>
      </mp>
        . . .
   </PointList>
  
   <Body mat='1' angle='0' thick='1' vx='1000' vy='0' conc='0' temp='0'>
      <RotateZ>20</RotateZ>
      <RotateY>atan(y/x)*180/3.14159</RotateY>
      <Oval units='mm' xmin='10' xmax='40' ymin='10' ymax='40'/>
      <Unrotate/>
      <Rect units='mm' xmin='10' xmax='40' ymin='10' ymax='40'/>     
   </Body>

   <Hole>
      <Rect xmin='20' xmax='30' ymin='20' ymax='30'/>     
   </Hole>

   <Body mat='1' angle='0' thick='1' vx='-1000' vy='0'>
      <Polygon>  
         <pt units='mm' x='60' y='10'/>  
         <pt x='90' y='10'/>
         <pt x='90' y='30'/>
         <pt x='75' y='40'/>
         <pt x='60' y='30'/>
      </Polygon>
   </Body>
   
   <Body mat='1' angle='0' thick='1' vx='1000' vy='0' vz='20' conc='0' temp='0'>
      <Sphere units='mm' xmin='10' xmax='40' ymin='10' ymax='40' zmin='0' zmax='10'/>     
      <Box units='mm' xmin='10' xmax='40' ymin='10' ymax='40' zmin='0' zmax='10'/>     
      <Cylinder units='mm' xmin='10' xmax='40' ymin='10' ymax='40' zmin='0' zmax='10' axis='3'/>     
      <Polyhedron>
          <faces style="15482637">
 1.496206e+002  2.879507e+000  1.775602e+002
 5.361129e+001  3.039185e+000  1.498799e+002
 1.496158e+002  2.209645e+000  1.775729e+002
 5.360652e+001  2.369322e+000  1.498926e+002
 1.551372e+002  2.477263e+000  1.584236e+002
 5.912789e+001  2.636940e+000  1.307433e+002
 1.551324e+002  1.807400e+000  1.584363e+002
 5.912311e+001  1.967077e+000  1.307560e+002
          </faces>
      </Polyhedron>
   </Body>

   <BMP name='GrayScaleFile.bmp' units='mm' width='5.64' height='4.51' angles='GrayAngles.bmp'>
      <Origin units='mm' x='10' y='20' z='0'/>
      <Intensity mat='1' imin='0' imax='120'>
         <vel units='mm/sec' x='100' y='200' z='0'/>
         <Thickness units='mm'>10</Thickness>
         <Angle>0</Angle>
         <Temperature>25</Temperature>
         <Concentration>0</Concentration>
      </Intensity>
      <Intensity imin='0' imax='255' minAngle='0' maxAngle='90'/>
      <RotateZ>20</RotateZ>
      <RotateY>atan(y/x)*180/3.14159</RotateY>
   </BMP>
</MaterialPoints>

where

<PointList>
A series of material points are defined with subordinate <mp> elements. The attributes give the element number that contains that point (elem - this attribute is optional; if not provided the element will be looked up), the material number (mat, see Defining Materials; matl can be used for backward compatibility and numbers can be replaced by names using a matname attribute as explained here), the material angle (angle in degrees which is used for some anisotropic materials), the thickness (thick in mm, 2D only), the initial concentration (conc set as potential from 0 to 1 where 1 is the saturation concentration for the particle's material type) when doing diffusion calculations, and the initial temperature (temp) when doing thermal loading or conduction calculations.
     Wherever conc is allowed, you can alternatively use wtconc to enter the actual weight fraction concentration rather than a concentration potential. The weight fraction concentration must be between 0 and the saturation concentration for the particle's material type. If both conc and wtconc are used, the one that appears last will determine the concentration.
     The elements subordinate to <mp> define the material point location (<pt> and its (x, y, (z if 3D)) coordinates in attributes in mm, but an optional units attribute can specify other length units) and the material point's initial velocity (<vel> and the velocity in the x, y, and z (if 3D) directions in mm/sec, but an optional units attribute can specify other velocity units). If not specified, the initial velocity, angle, and concentration potential will all be zero, the thickness (if 2D) will be 1 mm, and the temperature will be equal to the stress-free temperature.
     Normally the particle mass is determined from the size of the elements and the number of particles per element. When running a simulation based on material points from a previous simulation, a subordinate <mass> element can be used to set the particle mass (in g, but an optional units attribute can specify the mass units).
<Body>
Defines a block of material. All material points in the block will be assigned the same material number (mat, see Defining Materials; numbers can be replaced by names using a matname attribute as explained here), material angle (angle in degrees which is used for some anisotropic materials), thickness (thick in mm), initial velocities in the x, y, and z (if 3D) directions (vx, vy and vz in mm/sec), the initial concentration (conc set as potential from 0 to 1 where 1 is the saturation concentration for the particle's material type) when doing diffusion calculations, and the initial temperature (temp) when doing thermal loading or conduction calculations. A single <Body> group can contain any number of the shape commands below. 2D analyses use only 2D shapes, while 3D analyses use only 3D shapes. The shapes can overlap and the correct number of material points will be still be created. The particle properties will be for the first shape command that intersects that location. Material points created using <PointList> elements, however, will ignore points created by <Body> elements. The user is thus responsible for insuring no duplication of points at the same location when mixing <PointList> and <Body> elements. If not specified, the initial velocity, angle, and concentration potential will all be zero, the thickness (if 2D) will be 1 mm, and the temperature will be equal to the stress-free temperature.
<RotateZ>20</RotateZ>, <RotateY>y-z</RotateY>, <RotateX>45</RotateX>
These commands are to rotate the material axes in the current <Body> to orient anistropic materials in their initial configuration. You apply rotations in order by using one to three commands. There are two physical pictures. First, you can imagine sitting on the material coordinate system and defining a series of clockwise rotations about material axes that transform the x-y-z analysis axes into the material axes. Alternatively (and identically) the transformation can be viewed as counter-clockwise rotations about analysis axes that transform the material axes into the analysis axes. The text for each command is either an angle or a user-defined function that evaluates to an angle (either are in degrees).
     All single and double angle rotations are allowed (x, y, z, xy, xz, yx, yz, zx, and zy). When doing three rotations, the only schemes currently supported are zyx and zyz. It is easy to add more if needed (or to rearrange coordinates to make one of these work).
     2D calculations only allow z axis rotation, which can be done with either an angle attribute on the <Body> command or a single <RotateZ> command (although the <RotateZ> method is needed if you want to use a user-defined function). Also note that use of any Rotate command will override any number used in angle (i.e., angle will be ignored).
<Unrotate/>
This command cancels the rotations in the current <Body> section. Each shape command in the body use the current rotation transformation. Interspersing this command with new rotations allows each shape to have its own transformations.
<Hole>
Defines empty space within a body. A single <Hole> group can contain any number of the shape commands to define regions that are holes within bodies. 2D analyses use only 2D shapes, while 3D analyses use only 3D shapes. Any subsequent <Body> structures that overlap the hole will not create material points in the holes. The defined holes must come before the defined bodies that have the holes. Material points created using <PointList> elements, however, will ignore holes.

2D Shapes

<Oval>
Fills or empties the oval bounded by the parameters xmin, xmax, ymin, and ymax (an optional units attribute can specify the length units) with material points. The parameters be entered with numeric values or relative to the mesh edges.
<Rect>
Fills or empties the rectangle bounded by the parameters xmin, xmax, ymin, and ymax (an optional units attribute can specify the length units) with material points The parameters be entered with numeric values or relative to the mesh edges.
<Polygon>
Fills or empties the polygon drawn by the points specified in <pt> commands (point coordinates in x and y attributes. An optional units attribute can specify the length units.) with material points. The polygon can have any number of points (but requires at least 3). The last point is automatically connected to the first point to make a closed region.

3D Shapes

<Sphere>
Fills or empties the spheroid bounded by the parameters xmin, xmax, ymin, ymax, zmin, and zmax (an optional units attribute can specify the length units) with material points. The parameters be entered with numeric values or relative to the mesh edges.
<Box>
Fills or empties the box bounded by the parameters xmin, xmax, ymin, ymax, zmin, and zmax (an optional units attribute can specify the length units) with material points. The parameters be entered with numeric values or relative to the mesh edges.
<Cylinder>
Fills or empties the cylinder or cone bounded by the parameters xmin, xmax, ymin, ymax, zmin, and zmax (an optional units attribute can specify the length units) with material points. The axis attribute specifies the axis of the cylinder in the 1 (for x), 2 (for y) or 3 (for z) direction. The parameters (except axis) may be entered with numeric values or relative to the mesh edges.
    To fill or empty a cone, add a radius attribute with a value between -1 and 1. This value gives the relative radius of the cylinder at the top (if > 0) or at the bottom of the cylinder (if < 0) while the opposite end will have relative radius of 1. Entering 0 gives a cone with a point on the top (to get a cone with point on the bottom enter a small negative number). The relative radius varies linearly along the length on the cylinder.
<Polyhedron>
Fills or empties an arbitrary 3D shape specified by describing the object as a collection of triangular faces. The faces are specified with one or more faces commands subordinate to the Polyhedron command.
<faces style='tripts'>
The faces command has a set of options (specfied in the style attribute) to help in entering certain geometries. These options will grow as needed to solve new problems as they arise. For example, it would be easy to handle any number of connected triangular faces, but that option is not yet programmed. Here are the currently available options:
  • style='tripts' or style='trivectors': define a triclinic shape, which is an eight-corned box for which opposite faces are parallel, but the corners need not meet at right angles. For this style, you specify 4 points (or 12 coordinates). For tripts, the 4 points are the points labeled in the figure below. For trivectors, the first point is point 1 (the origin) and the remaining three points are the vectors between the shape's origin that point.
  • style='15482637': define an arbitrarily shaped 8-corner box. The provided numbers map the order of the provided 8 points (or 24 coordinates) to the standard order shown in the figure below.
Between the <faces> and the </faces> commands, all the data needed for the selected style are entered in a free format. The only requirements are the the correct number of valid numeric data points for the choosen style are provided and that the points are delimited by white space, commas, colons, and/or semicolons. The number of points per line does not matter and multiple delimiters between number can be used if desired.

BMP Shape

<BMP>
This tag reads the bit map file specified in the name (as full or relative path name) and assigns material points to the grid (or element properties in an FEA mesh) based on intensities within the bit map. The width and height attributes (an optional units attribute can specify the length units) define the size of the image. You can specify both a width and a height or just one dimension. In addition, if the number is positive it is total width or height, but if it is negative it is interpreted as the specified length unit per pixel. If only one dimension is given, the other will be calculated based on the image aspect ratio. You can specify dimensions that differ from the image to scale one direction differently than the other direction.

The image file must be a gray scale, non-compressed BMP file using 1 bit (2 shades, black and white), 4 bit (16 shades of gray), or 8 bit (256 shades of gray) resolution. Many programs can save graphics as BMP files.

Image files are always 2D. For 2D analysis, the lower-left corner of the image can be mapped to any point on the mesh using a subordinate <Origin> tag (x and y location of the origin with an optional units attribute and the default origion is (0,0)). For 3D analysis, a single image represents a slice at constant z coordinate specified in a z attribute of the <Origin> tag. The image will be mapped to x and y locations (relative to the specified origin) in a plane of constant z. A 3D shape can be built up by a series of BMP commands at various z locations such as data from X-ray CT slices.

Material points (or FEA element material types) will be assigned to various material types as determined by one or more subodinate <Intensity> blocks. The atttibutes on each <Intensity> block define the material number (mat, see Defining Materials; numbers can be replaced names using a matname attribute as explained here) and the minimum and maximum intensity for that material. The intensity is on a scale from 0 (for black) to 255 (for white). You can create a composite by having multiple shades of gray and corresponding multiple <Intensity> blocks to assign different shades to different materials. You can also build a model by superposing multiple <BMP> groups on the same block of space. The subsequent images will only fill space that was not occupied by prior images.

The subordinate <Thickness> (2D only), <Angle>, and <Temperature> let you set the thickness and any non-zero value for angle and temperature for the materal point (or FEA element). For MPM calculations additional, subordinate <vel> and <Concentration> let you set any non-zero values for velocity and concentration (set as a potential from 0 to 1 where 1 is the saturation concentration for the particle's material type). These properties function exactly like the corresponding attributes to the <Body> tag. The <Thickness> and <vel> settings can have an optional units attribute to specify the units. You can assign material points (or FEA elements) of the same material to different velocities, temperatures, etc., by having multiple shades of gray in the image and corresponding multiple <Intensity> blocks to assign different shades to different initial conditions. A particle (or FEA element) location with intensity not matching any <Intensity> block will create a hole or a location with no material point (or no FEA elements).

The BMP tag can have an optional angles attribute that specifies a second gray-scale file in which the shades of gray indicate the orientation angle for material points (or FEA elements). This angle can determine orientation of material points (or FEA elements) with anisotropic properties. The file must be exactly the same size (horizontal and vertical pixels) as the image file in the name attribute. To map the gray scale to angle, there should be an <Intensity> command with imin, imax, minAngle, and maxAngle attributes. This command defines a linear interpolation of gray scale intensity (0 for black to 255 for white) to material point angle through the points (imin, minAngle) and (imax, maxAngle) or 
angle = minAngle + ((i-imin)/(imax-imin)) * maxAngle
where i is intensity of the pixel. This function will be used for any intensity, which means it is not restricted to just imin < i < imax.

For MPM calculations, the BMP block can have optional <RotateZ>, <RotateY>, and <RotateX> commands to rotate the material axes for all particles defined by the BMP block for 2D (Z only) or 3D calculations (X, Y, and Z). There are thus three ways to set material angle in a BMP block:
  1. Use <Angle> command in an <Intensity> block - this option can only use a single constant (i.e., all angles for that material must be the same), but each material type can have a different angle.
  2. Use <RotateZ>, <RotateY>, and <RotateX> commands - these can set a constant angle or use a user-defined function that evaluates to an angle (in degrees). If any rotate commands are used, all <Angle> commands are ignored. Furthermore, all material types use the same angle settings (i.e., you cannot use different rotation angles or functions for each <Intensity> block).
  3. Use an angles attribute on the BMP tag with an angle definition file - if this third option is used, all <Angle> and rotation commands will be ignored.

Reading Material Points from a Previous Run

Sometimes it is useful to run a simulation, output the material point locations, and them use those settings for a new MPM simulation. One approach is to extract all material point locations to an XML file using the ExtractMPM tool, and then input that file to a new commands file. The process is:

  1. Run an MPM simulation
  2. Select an archive time and use ExtractMPM to extract particle data to an XML file. A typical command would be 
    ExtractMPM -Xh -q mass -o particles Archive.1843
    where Archive.1843 is desired time archive containing material point locations. The points will be written to an XML file called particles.xml. The output file will have one <mp> element for each material point and all those elements within a single <PointList>. See the ExtractMPM tool documentation for more details on an XML extraction.
  3. Finally, include this saved file in another commands file using an XML entity reference. First, define the path to the new file in the file's DOCTYPE element such as: 
    <!DOCTYPE JANFEAInput SYSTEM '/full path to/NairnMPM.dtd'
[   <!ENTITY mpfile SYSTEM "pointlist.xml">
]>
    Then, import that file in the <MaterialPoints> element such as 
    <MaterialPoints>
   &mpfile;
</MaterialPoints>
