Defining Materials

NairnFEA and NairnMPM support a variety of material types. An analysis can mix any number of materials. The materials are defined in a series of Material tags. A specific material is defined with:

<Material Type='1' Name='Polymer'>
     .
     .
     .
</Material>

where Type is the material type (integer) which defines the type of material and Name is any user-defined description of the the material (<200 characters and most be unique among defined materials if you reference them by name). The material properties for each type are defined by subordinate tags within the Material element.

When materials are assigned to FEA mesh elements, MPM materials points, MPM crack tips, or in a few other commands, they can be referenced by number or by name:

By Numbers
In this method, you use mat="#" attributes where # is the material number. The materials are defined by numbers in the order they appear in the output file with the first material being number 1. You have to be careful to use the correct number. If you add new materials to a file, it is best to add them to the end of the materials list, otherwise previous commands that referenced materials after an inserted material, will point to the wrong material.
By Name
In this method, you use matname="A Name" attributes where "A Name" is a unique material name that matches the name of any defined material in the file. You can use these names even before the materials are defined, but errors will occur if you reference materials that are not eventually defined or if the defined materials do not have unique names.

All defined materials will appear in the output file in the ordered referenced by name rather than in the order defined in the input file. But, if you do not reference any materials by name, they will appear in the order found in the input file. If some, but not all, materials are referenced by name, those that were not referenced will be output at the end of the materials in the output file by their order in the input file. If you mix material attributes by number and by name, the numbers uses have to refer to how they appear in the output file and not in the input file. For this reason, it is hard to mix the two methods and that approach should be avoided. Finally, if you use both mat and matname attributes in a single command, the matname one will be used and the mat will be ignored.

Certain materials work in NairnFEA and NairnMPM, while others are limited to NairnFEA or NairnMPM. Similarly, not all materials for NairnMPM can be used in 3D calculations. The 2D NairnMPM columns has two marks. The first means it works for planar MPM while the second indicates it works for axisymmetrix MPM. Material Properties for all Material Types

There are some material properties that are common to all material types. These properties are only used in NairnMPM and are not allowed in NairnFEA. The common properties are

<rho>1.5</rho>
The material density (in g/cm3) (can be used in NairnFEA files, but it is ignored).
<Cp>1.5</Cp>
The material constant-pressure heat capacity (in J/(kg-K)). Only used when doing conduction calculations. If only one of Cp and Cv are entered, they will both be set to the one entered value.
<Cv>1.5</Cv>
The material constant-volume heat capacity (in J/(kg-K)). Only used by some materials. If only one of Cp and Cv are entered, they will both be set to the one entered value
<csat>0.5</csat>
The material's saturation concentration entered as a weight fraction. Only used when doing diffusion calculations.
<Propagate criterion='1' direction='0' traction='1'/>
<AltPropagate criterion='7' direction='4' traction='1'/>
These properties let each material have a different primary and secondary propagation criterion and/or directions than the ones defined in the MPM Header. Set these criterion as described for the Propagate and AltPropagate commands in the MPM Header. To activate propagation:
  1. The MPM Header must have a Propagate command.
  2. The MPM Header can optionally have an AltPropagate command.
  3. Any material can override the default primary and/or secondary criteria by using a Propagate and/or AltPropagate command within its material defintion. Even if the MPM Header does not have an AltPropagate command, a material can add a secondary criterion with its own AltPropagate command.
<KIc>2.3</KIc>
Critical mode I stress intensity factor (in MPa m1/2). Only used when there is crack propagation with criteria 1, 4, or 5.
<KIexp>2.3</KIexp>
Exponent p in the elliptical criteria for crack growth. Only used for crack propagation with criterion 5.
<KIIc>1.2</KIIc>
Critical mode II stress intensity factor (in MPa m1.2). Only used when there is crack propagation with criterion 5.
<KIIexp>2.3</KIIexp>
Exponent q in the elliptical criteria for crack growth. Only used for crack propagation with criterion 5.
<JIc>2.3</JIc>
Critical mode I energy release rate (in J/m2). Only used when there is crack propagation with criteria 2, 3, or 7. For criterion 2, it is only used if the initTime is not being used. Also used by traction-law materials.
<JIIc>1.2</JIIc>
Critical mode II energy release rate (in J/m2). Currently only used by traction-law materials.
<nmix>2</nmix>
A parameter used in some mixed-mode failure laws (e.g., to implement failure when (GI/JIc)nmix + (GII/JIIc)nmix = 1 in TriangularTraction cohesive law).
<initTime>3</initTime>
Time (in ms) to start crack growht. Only used when there is crack propagation with criteria 2. For criterion 2, initTime takes precidence of a JIc property.
<speed>5</speed>
Define the constant crack speed (in m/sec). This speed, however, is only active when there is crack propagation with criterion 2.
<maxLength>20</maxLength>
Define the maximum length of the crack (in mm). The crack growth will stop when this length is reached. This limit, however, is only active when there is crack propagation with criterion 2.
<gamma>.5</gamma>
Only used when there is crack propagation with criterion 3. Still in development and not meant for general use.
<p>.5</p>
Only used when there is crack propagation with criterion 3. Still in development and not meant for general use.
<gain>10e5</gain>
Only used when there is crack propagation with criterion 3. Still in development and not meant for general use.
<xGrow>20</xGrow> and <yGrow>0</yGrow>
These properties (one or both with default 0) specify a unit vector for a constant crack growth direction. They only apply for crack propagation with criterion 2 and then only when that criterion selects the default propagation direction. The result is constant crack growth direction regardless of stress state or crack tip orientation. Any input vector will be normalized to a unit vector. If constant crack growth directions with a fixed crack are located precisely on grid lines, it is possible for the crack algorithm to not recognize the crack plane. It is better to move such a crack sligthly off grid lines.
<delIc>0.1</delIc> and <delIIc>0.1</delIIc>
Critical crack opening displacement. Only used when there is crack propagation by criterion 6. Also used by traction-law materials.
<color red='0.1' green='0.5' blue='0.3'/>
Sets the color of the material with red, green, and blue values between 0.0 and 1.0. The color may be used by post-analysis software, but it has no affect on the calculations.
<Friction mat='2'>0.3</Friction>
This command within a material definition allows multimaterial mode to have different contact laws between different pairs of materials. Set the contact law as explained for Friction command in the Cracks section of the MPM header except that a setting less than -10 means to assume materials are always in contact and revert to single material mode. The mat attribute defines contact mechanics between this material and the one listed (self contact is ignored). Alternatively, the other material can be specified by name using a matname attribute as explained here. If a node has more than two materials, the contact law used will be one between each material and the other material with the most volume.
    You only need to set friction on one of the materials in a pair. If you set both, the setting for the second material will override the one from the first material.

The material type is Type=3. Imperfect Interface Material

The material type is Type=5 and is used to model imperfect interfaces. The input properties are

<Dn>200</Dn>
Stiffness for loading the interface in the normal direction (in MPa/mm)
<Dt>20</Dt>
Stiffness for loading the interface in the tangential direction (in MPa/mm)

Imperfect interfaces allow displacement discontinuties along the interface. The discontinuity in the normal direction is σn/Dn and in the tangential direction is τt/Dn where σn is the normal component of the traction on the interface and τt is the tangential component. Interface parameters of Dn = Dt = ∞ leads to zero displacement discontinuity or a perfect interface; Dn = Dt = 0 leads to zero stress or a debonded interface. All other finite values define an imperfect interface. The interface is linear and thus compressive normal stress will lead to non-physical negative displacement discontinuties or interface overlap. If an interface is in compression, then Dn should be infinite, at least over regions of compressive stress.

Mooney-Rivlin Hyperelastic, Isotropic

The material type is Type=8. It is a large-deformation, isotropic, hyperelastic material. The stresses are defined from the strain energy, which is defined by invariants of the left Cauchy Green Tensor ([B] = [F][F]T where [F] is the deformation gradient). In a Mooney-Rivlin material, the strain energy depends on three material properties:

2W = G1(I1-3) + G2(I2-3) + K(J-1)2

where I1 = Trace([B])/J2/3, I2 = 0.5(I12 - [B]..[B]/J4/3), and J = Det([F]). The remaining parameters are material properties:

<G1>0.1</G1>
The G1 shear modulus (in MPa). The low-strain shear modulus is G1+G2.
<G2>0.33</G2>
The G2 shear modulus (in MPa). The low-strain shear modulus is G1+G2.
<K>2.5</K>
The K modulus (in MPa), which is equal to the low-strain bulk modulus.
<alpha>60</alpha>
The material thermal expansion coefficient (in ppm/C)
<beta>60</beta>
The material concentration expansion coefficient (in strain/wt fraction)
<D>400</D>
Diffusion constant (in mm2/sec). Only used when doing diffusion calculations.
<kCond>400</kCond>
Thermal conductivity (in W/(m-K)). Only used when doing conduction calculations. NairnMPM only.
Other Properties
See the properties common to all materials.

A neo-Hookean material is a special case of a Mooney-Rivlin material by setting G2 = 0.

History Data for Mooney-Rivlin Materials

This material use history #1 to store the volumetric strain (i.e., the determinant of the deformation gradient). The total strain, which is elastic, is stored in the elastic strain variable, but the plastic strain stores the left Cauchy Green tensor.

Von Mises Plastic, Isotropic

The material type is Type=9. The material is smaall-strain, isotropic and yields when f=0 according to the plastic potential

f = ||s|| - sqrt(2/3) * σY

where s is deviatoric stress. Since its magnitude, or ||s||, is a stress invariant, this yield condition is isotropic or independent of the material orientation angle. The material also allows isotropic work hardening based on cumulative equivalent plastic strain and implements either linear hardening or one non-linear hardening law. For linear hardening the yield stress is given by

σY = σY0 + Epα

where σY0 is the initial yield stress, Ep is the plastic modulus, and α is an internal variable that tracks cummulative, equivalent plastic strain (see history variable below). For nonlinear hardening the yield stress is given by

σY = σY0(1 + Kα)n

where K and n are dimensionless hardening parameters. The choice of linear or nonlinear hardening depends on which properties are defined.

The possible input properties are:

<yield>1.5</yield>
The initial yield stress (in MPa). This stress corresponds to the axial stress at yield during uniaxial, 3D loading.
<Ep>1.5</Ep>
The plastic modulus (in MPa) for linear hardening. This modulus is the slope of total stress as a function of plastic strain during uniaxial, 3D loading. The default is 0.0 which results in an elastic-perfectly plastic material or a material with no work hardening.
<ET>1.5</ET>
The tangent modulus (in MPa) for linear hardening. This modulus is the slope of total stress as a function of total strain after yielding in uniaxial, 3D loading. The key material property is Ep. Entering ET is provided as a convenience. If you enter it, Ep = E ET/(E-ET) will be calculated for you and override any entry of Ep.
<Khard>1500</Khard> and <nhard>0.29</nhard>
The dimensionless parameters for nonlinear hardening. If either one of these properties is provided, the material will use nonlinear hardening and any entries for Ep or ET will be ignored; otherwise the material will use linear hardening. Although linear hardening is a subset or nonlinear hardening (i.e., it is special case for n=1 and Ep=KσY0), it is more efficient to use actual linear hardening whenever n=1.
Other Properties
See Linear Elastic, Isotopic material.

History Data for Von Mises Plastic, Isotropic

The one history variable is the cummulative equivalent plastic strain, α, defined as the sum of sqrt(2/3)||dεp||. This variable can be archived as history variable 1.

Ideal Gas, as a Hyperelastic Material

The material type is an ideal gas, implemented as a large-deformation, isotropic, hyperelastic material at finite deformations. Its contitutive law is.

P = J-1 P0 T/T0

where J is determinant of the deformation tensor and J = V/V0. The pressure P is stored in the normal stresses or σxx = σyy = σzz = -P. The material can be used in 3D and 2D plain strain calculations, but not in 2D plane stress calculations.

The possible input properties are:

<P0>0.1013</P0>
The reference pressure (in MPa) at reference temperature and reference density. This must be a positive value greather than zero.
<T0>273.15</T0>
The reference temperature (in Kelvin)
<rho>0.00129</rho>
The reference density (in g/cm3) at reference temperature. This must be a positive value greather than zero.
Other Properties (where applicable)
See the properties common to all materials.

When using gas particles, the stress free temperature must be set to the desired temperature in Kelvin. The material properties refer to gas state at any reference conditions. For example, air might have reference pressure of 1 atm = 0.101325 MPa at any temperature T0. The density of air (or any gas) is then given by

rho = (P0/T0) * (Mg/R)

where Mg is the molecular weight of the gas (such as 28.97 g/mol for air) and R is the gas constant, R = 8.3144621 J/k/mol.

When thermal conduction is off, this material models isothermal compression and expansion. To model adibatic compression and expansion, turn conduction on and activate energy coupling. The energy will convert to heat resulting in heating during compression or cooling during expansion. See the Thermal command for details. There is no need to enter heat capacity because that term is calculated using ideal gas theory. You do, however, need to enter thermal conductivity. The current implementation uses a temperature independent conductivity, which may change in the future.

Ideal gas particles are fairly stable, but can be made unstable by certain boundary conditions on constraining walls. If stability problems arise, try different boundary conditions. They also do not work for irreversibale processes such as free expansion in to empty space. They are intended to be always bounded by pressure.

If the calculation archives strain energy, it tracks work done on the gas particles (in the form of P dV work).

The transition between the two states is determined by a dilation rule, a distortion rule, or a Von Mises stress rule (see transition property). By a dilation rule, the transition occurs when the volumetric strain (exx+eyy+ezz) reaches the entered critical value. By distortion rule, the transition occurs when the second strain invariant (sqrt(0.5 eij' eij') where eij' is deviatoric strain and repeated indices are summed) reaches a critical value. By a Von Mises stress rule, the transition occurs when the Von Mises stress (sqrt(0.5 sij' sij') where sij' is deviatoric stress and repeated indices are summed) reaches a critical value. When using a dilation rule, the new stress-strain relation can include a changed offset in volumetric strain corresponding to a stress-free conditions at a non-zero dilation relative to the initial state. This change normally leads to an instantaneous change in stress upon transition. When using a distortion or Von Mises stress rule, the offset is ignored and the change is only a change in slope of mechanical properties. The properties determining the transition are:

<transition>1</transition>
Transition is controlled by a dilation rule (if 1), a distortion rule (if 2), or Von Mises stress rule (if 3) (the default is dilation).
<critical>0.02</transition>
Critical value for the transition. The value is volumetric or deviatoric strain (in % for dilation or distortion rule) or stress (in MPa for Von Mises stress rule)..
<reversible/>
Make the transition reversible (the default is reversible).
<irreversible/>
Make the transition irreversible (the default is reversible).

Rigid Material

The material type is Type=11 and it is actually two materials in one, depending on the value for its SetDirection property:

  1. If SetDirection is 0 through 7, the material is not really a material and points that are this type of rigid material will not really be part of the MPM analysis. These rigid material points are used to set boundary conditions on the grid that move with the particles. The rigid material points should be initialized to a velocity, temperature, or concentration when they are created. As the rigid material points move through the grid, all nodes connected to those points will be assigned grid-based boundary conditions for the selected particle properties of velocity, temperature, and/or concentration. To have the boundary condition vary in time, use the the SettingFunction(2)(3) properties below to select the value rather than setting the initial particle properties (which can only be constant).
  2. If SetDirection is 8, the material will correspond to actual material points that move through the analysis. Since they are rigid, they will move with their prescribed velocity. They will interact with other material points only through multimaterial contact laws and thus this type of rigid material can only be used in multimaterial mode MPM calculations. These particles will also ignore grid-based velocity boundary conditions, including those set by rigid particles with SetDirection equal to 0 through 7. Rigid contact is briefly described in Lemiale et al. (2010).
        The advantage of this type of rigid material (unlike the other type) is that you can set frictional properties for contact between rigid materials and non-rigid materials. These rigid materials will move in the direction defined by the initial velocity. To have non-constant (e.g., rotational) motion, use the SettingFunction(2)(3) properties below rather than the initial partical velocity. If two different rigid materials of this type interact at the same node, however, there is a conflict and the calculation will stop. Finally, this type of rigid material can only set velocity; it cannot set temperature or concentration.
        This type of material ignores cracks, but that actually makes to possible to use them for new crack problems. For example, these rigid particles can provide wedge opening of a crack. If the rigid particles run along an explicit crack plane, those particles may contact particles on either side of the crack and then move the top and bottom surfaces of the crack apart. It may be the only way to wedge open a crack (i.e., the only material that can act like it is within a crack rather than on either side of the crack). All crack calculations (e.g., crack contact, traction laws, imperfect interfaces, updating of crack position, and J integral) will ignore the rigid particles. The interactions with the rigid particles arise only by contact laws between the rigid particles and the various crack velocity fields.

(1-D) = δpk*(δc-δmax)/(δmax*(δc-δpk))

where δmax is the maximum effective cod seen on the crack particle, except that it is initialized to the peak cod (δpk) at the start of the calculations. Once D is found, the normal and tangential tractions are Tn = keffδc and Tt = keffδt, which satisfy the effective traction - effective cod law given above. Note that during monotonic loading the traction will remain on the curve. If unloading occurs, however, the tractions follow a line back to the original for the peak cod.

Cubic Traction Law Material

The material type is Type=14. This material is a traction law material for crack surfaces and therefore can only be used on cracks or when creating new crack surfaces by the Propagate option. The traction laws in pure mode I or pure mode II each have cubic shape. These cubic laws have zero slope at the end (δc) and the peak stress is always at δc/3. The key properties are illustrated below:

You must enter properties for both normal opening (mode I) and shear (mode II) traction laws using the following parameters:

<sigmaI>10</sigmaI> and <sigmaII>100</sigmaII>
The peak stress or cohesive stress of the traction law (σ). Enter in units of MPa.
<JIc>10</JIc> and <JIIc>100</JIIc>
The toughness of the traction law or area under the traction law (Jc). Enter in units of J/m2.
<delIc>10</delIc> and <delIIc>100</delIIc>
The maximum opening displacement or critical displacement (δc) at which debonding occurs. Enter in units of mm.
<nmix>2</nmix>
The exponent used in the mixed-mode failure criterion (see below). Enter nmix<=0 to choose infinity, which is decoupled failure criterion.

You must then enter exactly two of the first three properties. These properties are interrelated and the third one will be calculated using Jc = 9000 σmax δc/16. The full traction law is:

σ = (27/4) σmax (δ/δc)(1-(δ/δc))2.

The failure criterion under possible mixed-mode conditions is identical to the one used in the TriangularTraction law.
