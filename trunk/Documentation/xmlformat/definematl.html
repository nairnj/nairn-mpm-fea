Defining Materials

NairnFEA and NairnMPM support a variety of material types. An analysis can mix any number of materials. The materials are defined in a series of Material tags and logically numbered beginning with 1. When materials are assigned to FEA mesh elements, MPM materials points, or MPM crack tips, they are referenced by this index number. A specific material is defined with:

<Material Type='1' Name='Polymer'>
     .
     .
     .
</Material>

where Type is the material type (integer) which defines the type of material and Name is any user-defined description of the the material. The material properties for each type are defined by subordinate tags within the Material element.

Certain materials work in NairnFEA and NairnMPM, while others are limited to NairnFEA or NairnMPM. Similarly, not all materials for NairnMPM can be used in 3D calculations (although more will be upgraded in the future). The material types and their supported uses are in the following table:

NameTypeDescriptionNairnFEA2D NairnMPM3D NairnMPM
Isotropic 1 Linear elastic, isotropic XXX
Transverse 1 2 Linear elastic, transversely isotropic with unique axis perpendicular to plane of the 2D analysis. XXX
Transverse 2 3 Linear elastic, transversely isotropic with unique axis in plane of 2D analysis. Unless rotated by "angle" options, the unique material axis is parallel to the y axis for 2D analyses and parallel to the z axis for axisymmetric analyses. XXX
Orthotropic 4 Linear elastic, orthotopic material XXX
Interface 5 A definition of an imperfect interface X
Viscoelastic 6 Linear viscoelastic material with sum of relaxation times XX
Mooney 8 Mooney-Rivlin non-linear elastic rubbery material X
VonMises 9 Elastic-plastic material with a VonMises stress yield criterion and isotropic hardening XX
HillPlastic 15 Elastic-plastic orthotopic material with a Hill yield criterion and hardening XX
MGSCGLMaterial 17 Isotropic material using the Mie-Grüneisen equation of state with shear modulus and yield stress from Steinberg, Cochran, and Guinan. XX

Bistable 10 Elastic, isotropic material with two stable states and reversible or irreversible transitions between the two states X
Rigid 11 Either a rigid material (infinite mass and stiffness) or a method to apply moving boundary conditions. XX
TriangularTraction 12 A mixed-mode cohesive law with triangular traction law for pure modes X

See the Material Technical Notes for details on implementation of these material types in NairnFEA and NairnMPM. These notes are mostly needed when working with source code. They also provide brief details about the mechanics for each material type.

Some materials have history-dependent data that can be archived (see MPM Header for details). Any calculated history data are listed at the end of each material type section. Besides the properties and history data specific to each type of material, there are some properties that are used for all types of materials. These common properties are listed in the next section.

Material Properties for all Material Types

There are some material properties that are common to all material types. These properties are only used in NairnMPM and are not allowed in NairnFEA. The common properties are

<rho>1.5</rho>
The material density (in g/cm3) (can be used in NairnFEA files, but it is ignored).
<Cp>1.5</Cp>
The material constant-pressure heat capacity (in J/(kg-K)). Only used when doing conduction calculations. If only one of Cp and Cv are entered, they will both be set to the one entered value.
<Cv>1.5</Cv>
The material constant-volume heat capacity (in J/(kg-K)). Only used by some materials. If only one of Cp and Cv are entered, they will both be set to the one entered value
<csat>0.5</csat>
The material's saturation concentration entered as a weight fraction. Only used when doing diffusion calculations.
<Propagate criterion='1' direction='0' traction='1'/>
This property lets each material have a different propagation criterion and/or direction than the ones defined in the MPM Header. Set the criterion as described for the Propagate command in the MPM Header.
<KIc>2.3</KIc>
Critical mode I stress intensity factor (in MPa m1/2). Only used when there is crack propagation with criteria 1, 4, or 5.
<KIexp>2.3</KIexp>
Exponent p in the elliptical criteria for crack growth. Only used for crack propagation with criterion 5.
<KIIc>1.2</KIIc>
Critical mode II stress intensity factor (in MPa m1.2). Only used when there is crack propagation with criterion 5.
<KIIexp>2.3</KIIexp>
Exponent q in the elliptical criteria for crack growth. Only used for crack propagation with criterion 5.
<JIc>2.3</JIc>
Critical mode I energy release rate (in J/m2). Only used when there is crack propagation with criteria 2, 3, or 7. For criterion 2, it is only used if the initTime is not being used. Also used by traction-law materials.
<JIIc>1.2</JIIc>
Critical mode II energy release rate (in J/m2). Currently only used by traction-law materials.
<nmix>2</nmix>
A parameter used in some mixed-mode failure laws (e.g., to implement failure when (GI/JIc)nmix + (GII/JIIc)nmix = 1 in TriangularTraction cohesive law).
<initTime>3</initTime>
Time (in ms) to start crack growht. Only used when there is crack propagation with criteria 2. For criterion 2, initTime takes precidence of a JIc property.
<speed>5</speed>
Define the constant crack speed (in m/sec). This speed, however, is only active when there is crack propagation with criterion 2.
<maxLength>20</maxLength>
Define the maximum length of the crack (in mm). The crack growth will stop when this length is reached. This limit, however, is only active when there is crack propagation with criterion 2.
<gamma>.5</gamma>
Only used when there is crack propagation with criterion 3. Still in development and not meant for general use.
<p>.5</p>
Only used when there is crack propagation with criterion 3. Still in development and not meant for general use.
<gain>10e5</gain>
Only used when there is crack propagation with criterion 3. Still in development and not meant for general use.
<xGrow>20</xGrow> and <yGrow>0</yGrow>
These properties (one or both with default 0) specify a unit vector for a constant crack growth direction. They only apply for crack propagation with criterion 2 and result in constant crack growth direction regardless of stress state. Any input vector will be normalized to a unit vector. If constant crack growth directions with a fixed crack are located precisely on grid lines, it is possible for the crack algorithm to not recognize the crack plane. Is it better to move such a crack sligthly off grid lines.
<delIc>0.1</delIc> and <delIIc>0.1</delIIc>
Critical crack opening displacement. Only used when there is crack propagation by criterion 6. Also used by traction-law materials.
<color red='0.1' green='0.5' blue='0.3'/>
Sets the color of the material with red, green, and blue values between 0.0 and 1.0. The color may be used by post-analysis software, but it has no affect on the calculations.
<Friction mat='2'>0.3</Friction>
This command within a material definition allows multimaterial mode to have different contact laws between different pairs of materials. Set the contact law as explained for Friction command in the Cracks section of the MPM header except that imperfect interface settings are not yet supported and setting less than -10 means to assume materials are always in contact and revert to single material mode. The mat attribute defines contact mechanics between this material and the one listed (self contact is ignored). If a node has more than two materials, the contact law used will be one between each material and the other material with the most volume.
    You only need to set friction on one of the materials in a pair. If you set both, the setting for the second material will override the one from the first material.

The material type is Type=3. The material is transversely isotropic with the axial direction of the material in the y direction of the x-y plane analysis. Imperfect Interface Material

The material type is Type=5 and is used to model imperfect interfaces. The input properties are

<Dn>200</Dn>
Stiffness for loading the interface in the normal direction (in MPa/mm)
<Dt>20</Dt>
Stiffness for loading the interface in the tangential direction (in MPa/mm)

Imperfect interfaces allow displacement discontinuties along the interface. The discontinuity in the normal direction is σn/Dn and in the tangential direction is τt/Dn where σn is the normal component of the traction on the interface and τt is the tangential component. Interface parameters of Dn = Dt = ∞ leads to zero displacement discontinuity or a perfect interface; Dn = Dt = 0 leads to zero stress or a debonded interface. All other finite values define an imperfect interface. The interface is linear and thus compressive normal stress will lead to non-physical negative displacement discontinuties or interface overlap. If an interface is in compression, then Dn should be infinite, at least over regions of compressive stress.

Von Mises Plastic, Isotropic

The material type is Type=9. The material is isotropic and yields when f=0 according to the plastic potential

f = ||s|| - sqrt(2/3) * σY

where s is deviatoric stress. Since its magnitude, or ||s||, is a stress invariant, this yield condition is isotropic or independent of the material orientation angle. The material also allows isotropic work hardening based on cumulative equivalent plastic strain and implements either linear hardening or one non-linear hardening law. For linear hardening the yield stress is given by

σY = σY0 + Epα

where σY0 is the initial yield stress, Ep is the plastic modulus, and α is an internal variable that tracks cummulative, equivalent plastic strain (see history variable below). For nonlinear hardening the yield stress is given by

σY = σY0(1 + Kα)n

where K and n are dimensionless hardening parameters. The choice of linear or nonlinear hardening depends on which properties are defined.

The possible input properties are:

<yield>1.5</yield>
The initial yield stress (in MPa). This stress corresponds to the axial stress at yield during uniaxial, 3D loading.
<Ep>1.5</Ep>
The plastic modulus (in MPa) for linear hardening. This modulus is the slope of total stress as a function of plastic strain during uniaxial, 3D loading. The default is 0.0 which results in an elastic-perfectly plastic material or a material with no work hardening.
<ET>1.5</ET>
The tangent modulus (in MPa) for linear hardening. This modulus is the slope of total stress as a function of total strain after yielding in uniaxial, 3D loading. The key material property is Ep. Entering ET is provided as a convenience. If you enter it, Ep = E ET/(E-ET) will be calculated for you and override any entry of Ep.
<Khard>1500</Khard> and <nhard>0.29</nhard>
The dimensionless parameters for nonlinear hardening. If either one of these properties is provided, the material will use nonlinear hardening and any entries for Ep or ET will be ignored; otherwise the material will use linear hardening. Although linear hardening is a subset or nonlinear hardening (i.e., it is special case for n=1 and Ep=KσY0), it is more efficient to use actual linear hardening whenever n=1.
Other Properties
See Linear Elastic, Isotopic material.

History Data for Von Mises Plastic, Isotropic

The one history variable is the cummulative equivalent plastic strain, α, defined as the sum of sqrt(2/3)||dεp||. This variable can be archived as history variable 1.

Bistable, Isotropic Elastic

The material type is Type=10. This material has two isotopic states. The first state (state "0") must be fully defined. Any property omitted for the second state (state "d") will be assumed to be indentical to the first state. The second state can optionally have an offset volumetric strain to give the deformed state a different zero-stress intercept than the undeformed state. The mechanical properties in the two states are set with:

<G0>0.5</G0>
Shear modulus in MPa of initial state.
<alpha0>60</alpha0>
Thermal expansion coefficient in ppm/degree C of initial state.
<beta0>60</beta0>
Concentration expansion coefficient in strain/wt fraction of initial state
<D0>20</D0>
Diffusion constant in initial state (in mm2/sec). Only used when doing diffusion calculations.
<kCond0>20</D0>
Thermal conductivity in initial state (in W/(m-K)). Only used when doing thermal conductivity calculations.
<Kd>1.0</Kd>
Bulk modulus in MPa of transformed state.
<Gd>0.2</Gd>
Shear modulus in MPa of transformed state.
<alphad>80</alphad>
Thermal expansion coefficient in ppm/degree C of transformed state.
<betad>60</betad>
Concentration expansion coefficient in strain/wt fraction of transformed state
<Dd>400</Dd>
Diffusion constant in transformed state (in mm2/sec). Only used when doing diffusion calculations.
<kCondd>20</D0>
Thermal conductivity in transformed state (in W/(m-K)). Only used when doing thermal conductivity calculations.

Other Properties
See the properties common to all materials.

The transition between the two states is determined by a dilation rule, a distortion rule, or a Von Mises stress rule (see transition property). By a dilation rule, the transition occurs when the volumetric strain (exx+eyy+ezz) reaches the entered critical value. By distortion rule, the transition occurs when the second strain invariant (sqrt(0.5 eij' eij') where eij' is deviatoric strain and repeated indices are summed) reaches a critical value. By a Von Mises stress rule, the transition occurs when the Von Mises stress (sqrt(0.5 sij' sij') where sij' is deviatoric stress and repeated indices are summed) reaches a critical value. When using a dilation rule, the new stress-strain relation can include a changed offset in volumetric strain corresponding to a stress-free conditions at a non-zero dilation relative to the initial state. This change normally leads to an instantaneous change in stress upon transition. When using a distortion or Von Mises stress rule, the offset is ignored and the change is only a change in slope of mechanical properties. The properties determining the transition are:

<transition>1</transition>
Transition is controlled by a dilation rule (if 1), a distortion rule (if 2), or Von Mises stress rule (if 3) (the default is dilation).
<critical>0.02</transition>
Critical value for the transition. The value is volumetric or deviatoric strain (in % for dilation or distortion rule) or stress (in MPa for Von Mises stress rule)..
<reversible/>
Make the transition reversible (the default is reversible).
<irreversible/>
Make the transition irreversible (the default is reversible).

History Data for Bistable, Isotropic Elastic Materials

History variable 1 will be 0 for the initial state and 1 for the deformed state after a transition.

Rigid Material

The material type is Type=11 and it is actually two materials in one, depending on the value for its SetDirection property:

  1. If SetDirection is 0, 1, 2, 3, or 4, the material is not really a material and points that are this type of rigid material will not really be part of the MPM analysis. These rigid material points are used to set boundary conditions on the grid that move with the particles. The rigid material points should be initialized to a velocity, temperature, or concentration when they are created. As the rigid material points move through the grid, all nodes connected to those points will be assigned grid-based boundary conditions for the selected particle properties of velocity, temperature, and/or concentration. To have the boundary condition vary in time, use the the SettingFunction property below to select the value rather than setting the initial particle properties.
  2. If SetDirection is 8, the material will correspond to actual material points that move through the analysis. Since they are rigid, they will move with their prescribed velocity. They will interact with other material points only through multimaterial contact laws and thus this type of rigid material can only be used in multimaterial mode MPM calculations. These particles will also ignore grid-based velocity boundary conditions, including those set by rigid particles with SetDirection equal to 0 through 4.
        The advantage of this type of rigid material (unlike the other type) is that you can set frictional properties for contact between rigid materials and non-rigid materials. These rigid materials will move in the direction defined by the initial velocity. They will either move at the magnitude of that velocity or the magnitude can varied by using the SettingFunction property below. If two rigid materials of this type moving at different velocities interact at the same node, however, there is a conflict and the calculation will stop. Finally, this type of rigid material can only set velocity; it cannot set temperature or concentration.
        This type of material ignores cracks, but that actually makes to possible to use them for new crack problems. For example, these rigid particles can provide wedge opening of a crack. If the rigid particles run along an explicit crack plane, those particles may cross through the cracks and then move the top and bottom surfaces of the crack apart. It may be only way to wedge open a crack (i.e., the only material that can act like it is within a crack rather than on either side of the crack). All crack calculations (e.g., crack contact, traction laws, imperfect interfaces, updating of crack position, and J integral) will ignore the rigid particles. The interactions with the rigid particles arise only by contact laws between the rigid particles and the various crack velocity fields.

The rigid material properties determine which boundary conditions are set:

<SetDirection>1</SetDirection>
Set to 1 to set x direction velocity, 2 to set y direction velocity, or 4 to set z direction velocity. The velocities in the selected direction will be set to that component of constant particle velocity. If the direction is set to 3, the velocity will be set in the direction of the net particle velocity, which may be in any direction in the x-y plane. To create the alternate type of rigid particles that can implement frictional contact, set this value to 8.
<SetTemperature/>
Set temperature to the particle temperature. This setting has no affect unless the analysis is doing thermal calculations. Can only be used if SetDirection is 0 through 4.
<SetConcentration/>
Set concentration to the particle concentration entered as a potential from 0 to 1 where 1 is the saturation concentration for the particle's material type. This setting has no affect unless the analysis is doing diffusion calculations. Can only be used if SetDirection is 0 through 4.
<SettingFunction>50*(1-sign(t-10))</SettingFunction>
By entering a user-defined function of time (t in ms and the function must depend only on time and cannot depend on position), the rigid particle setting can vary with time. When setting a direction, temperature, or concentration, the function should evaluate to the velocity in mm/s, temperature, or concentration potential. The initial velocity, temperature, or concentration assigned to the rigid particles when they are created will be ignored. The one exception is when SetDirection is 3 (for particle velocity set in a skewed x-y direction) or 8 (for rigid particles that can do contact). For these settings, the initial particle x and y velocities (for SetDirection=3) or complete velocity vector (for SetDirection=8) will define the velocity direction and the setting function will determine the magnitude (absolute value of setting function will be taken so rigid particle cannot reverse directions). Note that if the initial particle velocity is zero or the setting function ever becomes zero when SetDirection is 3 or 8, the particle will halt even if the setting function becomes non-zero at later times. Thus, when SetDirection is 3 or 8, the setting function must never evaluate to zero (unless you want it to stop).
     Since there is only one setting function, a rigid material with a setting function can set only one property and will set all particles of that material type to the same value determined by the function. To have rigid particles move with different values determined by a function, even if in the same direction, thus requires definition of multiple rigid material types.

Note that multimaterial mode MPM calculations can combine moving boundary condition rigid particles (SetDirection=0 to 4) with contact rigid particles (SetDirection=8) to provide different functions, but single-material mode MPM can only use boundary condition rigid particles (SetDirection=0 to 4).

Note that creating rigid particles but not setting any velocity direction (SetDirection=0) can be used to create moving temperature or concentration boundary conditions that move over the particles; i.e., the rigid particles will move through the real particles. Such moving temperature or concentration boundary conditions can only move at a constant velocity defined by the initial particle velocity.

If a rigid particle is added to a problem, but it does not set any velocity, temperature, or concentration (SetDirection=0 or omitted and SetTemperature and SetConcentration omitted), it will have no affect on the calculations. It may plot in visualization software and therefore appear to be part of the simulation, but it will be identical to the results that would be obtained if it had never been added to the problem.

Triangular Traction Law Material

The material type is Type=12. This material is a traction law material for crack surfaces and therefore can only be used on cracks or when creating new crack surfaces by the Propagate option. The traction laws in pure mode I or pure mode II each have the following triangular shape:

You must enter properties for both normal opening (mode I) and shear opening (mode II) traction laws using the following parameters:

<sigmaI>10</sigmaI> and <sigmaII>100</sigmaII>
The peak stress or cohesive stress of the traction law (σ). Enter in units of MPa.
<JIc>10</JIc> and <JIIc>100</JIIc>
The toughness of the traction law or area under the traction law (Jc). Enter in units of J/m2.
<delIc>10</delIc> and <delIIc>100</delIIc>
The maximum opening displacement or critical displacement (δc) at which debonding occurs. Enter in units of mm.
<nmix>2</nmix>
The exponent used in the mixed-mode failure criterion (see below). Enter nmix<=0 to choose infinity, which is decoupled failure criterion.
<kIe>200</kIe> and <kIIe>100</kIIe>
The slope of the initial elastic portion of the tractional law (k). Enter in units of MPa/mm.
<delpkI>0.1</delpkI> and <delpkII>0.2</delpkI>
The displacement at the location of the peak in the triangular law. Enter in dimensionless units as fraction of delIc or delIIc (0 to 1). You can enter zero to get pure linear softening. Although a pure linear softening law seems to work well, it is possible the effectively infinite stiffess in the beginning might cause issues with dynamic convergence of the results, especially is the cohesive stress is high. If convergence problems occur, you can increase these values to make the initial stiffness finite.

You must then enter exactly two of the first three properties. These properties are interrelated and the third one will be calculated using Jc = 500 σmax δc (in the entered units). You must enter zero or one for the last two properties. If one is provided, the other will be calculated from k = σmax/(δpeak δc). A valid traction law requires δpeak ≤ 1. If neither k nor δpeak are entered, the peak will be located at δpeak = 0.225926299. This location provides the closest match to the cubic traction law for area under the curve as a function of δ.

These entered traction laws give mode I or mode II traction law for pure mode I or pure mode II loading. For mixed-mode loading, the normal and and shear tractions still depend only on normal and shear displacements, respectively, but the conditions for failure change. Under pure mode loading, failure occurs with the relevent displacement reaches the critical displacement. For mixed-mode loading, failure occurs when:

(GI/JIc)nmix + (GII/JIIc)nmix = 1

where GI and GII are areas under mode I and mode II traction laws up to current displacements. Now this failure criterion will mean that failure may occur when normal or shear tractions are nonzero and result in a sudden drop to zero traction. This approach is identical to the one adopted by Thouless et. al to handle mixed-mode loading and appears to work well. There has been work to modify the traction laws so they always approach zero at time of failure (e.g., work of Hogberg (2006) or Tvergaard and Hutchinson (1994)). These approaches were tried, gave poor results for certain problems, led to non-physical coupling between mode I and mode II, and therefore were abandoned.

2F = (1/yldyy2) + (1/yldzz2) - (1/yldxx2)

( (1/yldii2) - (1/yldjj2) )2 ≤ 1/yldkk2 ≤ ( (1/yldii2) + (1/yldjj2) )2

In more practical terms, if two yield stresses are related by some ratio R = yldii/yldjj, then the third yield stress is bracketed by:

yldii/|1-R| ≤ yldkk ≤ yldii/(1+R)

Enter density, ρ, and heat capacity, CV, in properties common to all materials.

In this material, the pressure is related to the volumetric compression strain (x = -ΔV/V0) by the Mie-Grüneisen equation of state:

P = k1 x + k2 x2 + k3 x3 + γ0ρCV(T-T0)

where k1, k2, and k3 are calculated from input parameters:

k1 = ρC02 - γ02ρCVT0
k2 = 2ρC02S1 - (1/2)γ03ρCVT0
k3 = 3ρC02S12 - (1/3)ρC02γ0S1 + 2ρC02S2C0 - (1/6)γ04ρCVT0

Here T0 is the reference temperature for the material. The properties depend on the reference temperature, but the reference temperature is set using the StressFreeTemp command in the MPM header and not in the material. The k1 term is the low-strain, bulk modulus.

The shear modulus of the material is given by

G = G0 [ 1 + (Gp'/G0)(P/η1/3) + (GT'/G0)(T-T0) ]

where η = 1/(1-x).

This material yields when f=0 according to the plastic potential

f = ||s|| - sqrt(2/3) * σY

where ||s|| is the deviatoric or flow stress and the yield stress is given by the hardening law:

σy = σ0 (1 + β εpn) [ 1 + (Gp'/G0)(P/η1/3) + (GT'/G0)(T-T0) ]

where εp is the equivalent plastic strain. The yield stress is limited to the maximum yield stress. See D. J. Steinberg S. G. Cochran, and M. W. Guinan, "A constitutive model for metals applicable at high strain rates" (J. Appl. Phys., 51, 1498-1504 (1989)) for details.

Note that plasticity is implemented using J2 flow theory, which is a theory that assumes f depends on ||s|| and is independent of pressure. The yield stress, however, has a pressure-dependent term (see (Gp'/G0) term). The effect may be small and it appears that other implementions of this material take this same approach.

This material automatically has thermal expansion. You do not need to enter a thermal expansion coefficient because it is determined by other properties through:

α = (ργ0CV)/(3k1)

This material allows isotropic conduction and diffusion by entering isotropic transport properties (kCond, D, and beta) and bulk properties (Cp and csat).
