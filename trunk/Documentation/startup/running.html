Running NairnFEA and NairnMPM

Details on the running the code can be found in the OSUPDocs wiki. This section briefly mentions running using on a command line.

  • Running NairnFEA on a Command Line
  • Running NairnMPM on a Command Line

Running NairnFEA on a Command Line

You can run NairnFEA from a Unix or Linux command line using: 

NairnFEA [options] input > output

where

[options]
The options may be one or more of the following:
-a
Read the input file, set up the mesh and all boundary conditions and then abort before any calculations.
-H
Display help message.
-np 4
Set the number of processors to use for parallel execution.
-v
Validate the input XML using the DTD file specified in the !DOCTYPE line in the preamble of the input file.
-w
Set current working directory for resolving relatice path names (default is to use the input file).
-?
Display help message.
input
The path name of the input XML file.
output
The name of the output text file.

Running NairnMPM on a Command Line

You can run NairnMPM from a Unix or Linux command line using: 

NairnMPM [options] input > output

where

[options]
The options may be one or more of the following:
-a
Start the analysis, archive the initial conditions, and then abort the calculations.
-H
Display help message.
-np 4
Set the number of processors to use for parallel execution.
-r
Reverse the bytes when writing results to archive files. This option can be useful when you are running calculations on one computer, but visualizing them on a different computer which uses a different byte order.
-v
Validate the input XML using the DTD file specified in the !DOCTYPE line in the preamble of the input file.
-w
Set current working directory for resolving relatice path names (default is to use the input file).
-?
Display help message.
input
The path name of the input XML file.
output
The name of the output text file.
