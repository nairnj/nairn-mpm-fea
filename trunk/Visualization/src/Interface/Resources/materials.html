Material Types

A material is defined in scripting commands using a block that allows you to specify a new material type and define all its properties. The command block format is:

Material #1,#2,#3
  (Property commands - one per line)
Done

where #1 is the ID or Label for the material created by the command. #2 is the material name entered as an string. When other commands refer to this material, they should use parameter #1 (the ID or Label) and not the name. The full name in parameter #2, however, will appear in output results. #3 defines the type of material, which can be entered by name or number (see table below).

The Material command should be followed by lines that contain the material properties. Each property should be on a separate line and when the last property is specified, enter a line with the single Done Command. The various material types expect different properties. See the code project documentation on materials to see the properties expected for each material. To set a property, use the same name as the XML command name (case sensitive) follwed by the value for its argument. For example, to set modulus of isotropic material, use

E 2300

for the equivalent of <E>2300</E> as an XML command.

The following table list materials supported by NairnFEA or NairnMPM. The addition of new materials is an active development area, expecially for NairnMPM. When a material is supported by NairnFEAMPMViz, you can enter it using a Material block and scripting commands. Those materials are highlighted below. For all other materials, you hae to define them using an XMLData command (see commands help on that command for details). The boxes on the right tell which code supports each material type. For 2D MPM, the first X means it can be used in planar analyses and the second means axisymmetric analyses.
