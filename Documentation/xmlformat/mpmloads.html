Particle Boundary Conditions

Particle-based boundary conditions are used to set force, traction, and heat flux, and concentration flux conditions. The details are documented in the OSUPDocs wiki. This pages outlines their use.

All particle-based boundary conditions must be set up within a single <ParticleBCs> element. The format is

<ParticleBCs>
   (one ot more particle BC shape commands)
   
   <LoadBCs>
      ⋮
   </LoadBCs>
</ParticleBCs>

There are two ways to specify particle boundary conditions. The most common approach is to generate conditions using one or more shape commands (like <LdRect> above) to select particles and apply specified load, traction, heat flux, and/or concentration flux boundary conditions to all selected particles. The other way is to explicitly list each particle. This method is shown in the <LoadBCs> section above, is limited to load conditions.

Particle BC Shape Commands

These commands define shapes. All particles contained within the shape will be assigned boundary conditions for load, traction, heat flux, and/or concentration flux that are specified by elements subordinate to the shape command:

  • <BCLine x1='0' y1="0" x2='100' y2='0' tolerance='0.1'>
    ⋮
</BCLine>
    Deine a 2D line from (x1,y1) to (x2,y2) assign boundary conditions to all particles within optional tolerance of the line (in length units or set by an optional units attribute). The coordinates can alternatively be specified relative to the mesh edges.
  • <BCArc x1='0' y1='0' x2='100' y2='0' start='0' end='90' tolerance='0.1'>
    ⋮
</BCArc>
    Deine a 2D arc along an elipse inside the rectangle with corners (x1,y1) to (x2,y2) and assign boundary conditions to all particles within optional tolerance of the arc (in length units or set by an optional units attribute). The arc will trace the oval defined by the points over angles from start to end in the counter-clockwise direction where start=0 is along the x axis. The coordinates can alternatively be specified relative to the mesh edges.
  • <LdRect xmin='18.5' xmax='20' ymin='-12' ymax='12'>
    ⋮
</LdRect>
    Deine a 2D rectangle with ranges xmin to xmax and ymin to ymax and assign boundary conditions to all particles within that rectangle (in length units or set by an optional units attribute). The coordinates can alternatively be specified relative to the mesh edges.
  • <BCBox xmin='18.5' xmax='20' ymin='-12' ymax='12' zmin='0' zmax='10' axis='0'>
    ⋮
</BCBox>
    Deine a 3D box with corners (xmin,ymin,zmin) and (xmax,ymax,zmax) and assign boundary conditions to all particles within that box (in length units or set by an optional units attribute). The optional axis attribute can convert the shape to the cylinder enclosed within the box with axis along the specified direction (as x, 1, y, 2, z, or 3). The coordinates can alternatively be specified relative to the mesh edges.

Generated Particle Load or Traction Conditions

Particle load or traction conditions subordinate to shape elements are created using: 

   <LoadBC dir='1' style='1' load='400' time='0.0' function='x*t'/> 

   <TractionBC dir='1' style='1' face='1' stress='400' time='0.0' function='x*t'/>

where

  • dir - defines the loading direction (1 for x axis, 2 for y axis, 3 for z axis (if 3D), 1 for R axis (if axisymemtric), or 2 for Z axis (if axisymemtric)). For TractionBC, dir can be 11 or 12 to mean normal or shear traction relative the the selected face, respectively. A normal traction is positive if pointing out of the particle domain and a shear traction is positive when oriented in the counter-clockwise direction. Shear tractions are not allowed in 3D calculations.
  • style - defines the type of loading (see below).
  • face (for TractionBC) - the face of the original particle domain that is loaded by traction. In 2D, the faces are the edges numbered 1 through 4 in the counter-clockwise direction with the bottom edge being number 1. In 3D, 1 to 4 are the same faces (i.e., faces with normals (0,-1,0), (1,0,0), (0,1,0), and (-1,0,0) on the original particle domain, respectively), 5 is the bottom (with normal (0,0,-1)), and 6 is the top (with normal (0,0,1)).
  • load and time (for LoadBC) - defines the two parameters (value and time) for the specified style (see below). The load parameter is in force units or force/alt time units.
  • stress and time (for TractionBC) - defines the two parameters (value and time) for the specified style (see below). The stress parameter is in pressure units or pressure/alt time units. To calculate traction stresses, the simulation needs to know the grid thickness and it should match the particle thickness.
  • function - when using style=6, enter an arbitrary user-defined function of time (t in alt time units), of current particle position (x, y, z, D, T, R, and Z). in length units), and/or of current clockwise particle rotation angle (q in radians). The function should evaluate to load in force units (for LoadBC) or stress in pressure units (for TractionBC).

LoadBC options

  • <net/>
<perParticle/>
    By default, the load argument or the load returned by a function give the load on each particle with that boundary condition. Alternatively, a load or function can evaluate to the total force applied to all particles in the current shape. To enter total load, include a <net/> command within the shape command containing the LoadBC command before the LoadBC command. To specify load per particle, include <perParticle/> (or have no command since per particle is the default setting). These commands only apply to LoadBC commands.

Boundary Condition Styles

The value, style, and time attributes for particle-based boundary conditions are used to set the time-dependence of the boundary conditions. The style options are listed below and any number of options can be combined on a single particle. When a single particle has multiple conditions, the resulting condition will be a superposition of the imposed conditions.

Style=1 (constant)
Set condition to (value) starting at time in (time) attribute where (time) is in alt time units
Style=2 (linearly increasing condition)
Set condition to (value)*[t-(time)] where t, which is the current time, and (time) are in alt time units. The condition starts at time in the (time) attribute.
Style=3 (sine variation)
Set condition to (value)*sin[t*(time)] where t is current time (in ms) and (time) has units 1/ms.
Style=4 (cosine variation)
Set condition to (value)*cos[t*(time)] where t is current time (in alt time units) and (time) has units 1/(alt time) units.
Style=5 (silent boundary)
Set the boundary to absorbing boundary conditions. The intent of a silent boundary is to eliminate reflected waves at a free edge and mimic a sample in a semi-infinite domain. The silent boundary must have normal vector in x or y direction . The dir attribute specifies the direction of the normal vector. There is no need to specify a load or time argument. This boundary condition currently only works for linear elastic, isotropic materials and it requires a regular grid. Silent boundary conditions are only allowed for particle load and flux conditions.
Style=6 (user-defined function)
Set condition to the evaluated value of a user-defined function of time (t in alt time units) and/or nodal coordinates (x, y, z, R, Z, D, and T in length units). If the time attribute is used the condition starts at time in the (time) attribute and the function is evaluated at [t-(time)] (in alt time units). A function of rotational angle is sometimes needed for large displacement calculations.

Generated Heat Flux Conditions

There are two options for setting non-zero heat flux conditions, which are determined by the dir attribute (note: the use of heat flux conditions requires that you set the grid thickness to match the boundary particle's thickness):

Externally Applied Flux

To apply an external flux, use: 

    <HeatFluxBC dir='1' face='1' style='1' value='0' time='0.0' function='sinh(t)'/>

where

  • dir - must be 1 to indicate an external flux.
  • face - the face of the original particle domain that in on the boundary and has the concentration flux. In 2D, the faces are the edges numbered 1 through 4 in the counter-clockwise direction with the bottom edge being number 1. In 3D, 1 to 4 are the same faces (i.e., faces with normals (0,-1,0), (1,0,0), (0,1,0), and (-1,0,0) on the original particle domain, respectively), 5 is the bottom (with normal (0,0,-1)), and 6 is the top (with normal (0,0,1))
  • style - defines the type of flux using standard styles. The silent option has a special meaning and is used to set natural flux determined by (k grad T).n on particles on the edge with the boundary condition. It should enable simulation of heat flow through materials.
  • value and time - these arguments are used with the chosen style. The value should be in heat flux units or in heat flux/alt time units. Positive flux is into the object and negative is out of the object.
  • function - when using style=6, enter an arbitrary user-defined function of time (t in alt time units), of current particle position (x, y, z, D, T, R, and Z in length units), and/or of current clockwise particle rotation angle (q in radians). The function should evaluate to the flux in heat flux units.

Coupled Surface Heat Flux

This option sets the surface heat flux to some function of the current particle temperature: 

    <HeatFluxBC dir='2' face='3' style='6' function='254*(300^4-t^4)'/>

where

  • dir - must be 2 to indicate a coupled surface flux.
  • face - the face with the flux is as defined above.
  • style - must be 6 meaning the condition is specified in a function (see below)
  • function - enter an arbitrary user-defined function of t, where t stands for the current particle temperature (in K), of current particle position (x, y, z, D, T, R, and Z in length units), and/or of current clockwise particle rotation angle (q in radians). The function should evaluate to the flux in heat flux units. Positive flux is into the object and negative is out of the object.

Generated Concentration Flux Conditions

There are two options for setting non-zero flux conditions, which are determined by the dir attribute (note: the use of concentration flux conditions requires that you set the grid thickness to match the boundary particle's thickness):

Externally Applied Flux

To apply an external flux, use:

    <ConcFluxBC dir='1' face='1' style='1' value='0' time='0.0' function='sinh(t)'/>

where

  • dir - must be 1 to indicate an external flux.
  • face - the face of the original particle domain that in on the boundary and has the concentration flux. In 2D, the faces are the edges numbered 1 through 4 in the counter-clockwise direction with the bottom edge being number 1. In 3D, 1 to 4 are the same faces (i.e., faces with normals (0,-1,0), (1,0,0), (0,1,0), and (-1,0,0) on the original particle domain, respectively), 5 is the bottom (with normal (0,0,-1)), and 6 is the top (with normal (0,0,1)).
  • style - defines the type of flux using standard styles. The silent option has a special meaning and is used to set natural flux determined by (D grad c).n on particles on the edge with the boundary condition. It should enable simulation of solvent flow through materials.
  • value and time - these arguments are used with the chosen style. The value should be in solvent flux units or in solvent flux/alt time units. Positive flux is to into the object and negative is out of the object.
  • function - when using style=6, enter an arbitrary user-defined function of time (t in alt time units), of current particle position (x, y, z, D, T, R, and Z in length units), and/or of current clockwise particle rotation angle (q in radians). The function should evaluate to the flux in solvent flux units. When using a function, the value attribute is ignored, but time can be used to determine when the function is evaluated.

Coupled Surface Flux

This option sets the surface flux to some function of the difference between the particle concentration potential and the concentration potential of a surrounding reservoir: 

    <ConcFluxBC dir='2' face='3' style='6' function='t^2'  bath='1'/>

where

  • dir - must be 2 to indicate a coupled surface flux.
  • face - the face with the flux is as defined above.
  • style - must be 6 meaning the condition is specified in a function (see below)
  • function - enter an arbitrary user-defined function of t, where t now gives the difference between the particle concentration potential and the reservoir concentration potential, of current particle position (x, y, z, D, T, R, and Z in length units), and/or of current clockwise particle rotation angle (q in radians). The function should evaluate to the flux in solvent flux units. The sign is irrelevant. The applied flux will be positive if the particle concentration is less than the bath or negative if it is greater than the bath. The magnitude of the function gives the magnitude of the flux.
  • bath - the concentration potential (0 to 1) of the surrounding reservoir bath. Note that the bath is specified as a concentration potential (c/csat) and not actual weight fraction concentation.

Explicit Particle Loading

Explicit load boundary condition are defined with <load> elements within a single <LoadBCs> element: 

   <load pt='398' dof='2' style='1'>(data)</load>

where

  • pt - the material point number to be loaded.
  • (data) - the data in the element has two numbers whose meanings are the value and time arguments for the boundary condition styles. The value is in force units or force/alt time units and the time is in alt time units or 1/alt time units. When using an arbitrary function (see Style=6 below) the (data) in the element should be the a user-defined function of t (in alt time units), of current particle position (x, y, z, D, T, R, and Z in length units), and/or of current clockwise particle rotation angle (q in radians), that evaluates to load in force units.
  • dof - determines if applied load is in the x direction (dof=1), the y direction (dof=2), z direction (dof=3, 3D only), R direction (dof=1, if axisymnmetric), or Z direction (dof=2, if axisymmetric).
  • style - determines the time dependence of the boundary condition. The possible styles are defined above.
