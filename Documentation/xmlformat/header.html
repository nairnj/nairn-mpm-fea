Header

The main header is docummented in the on-line wiki for MPM and FEA input files. The main header is defined in the following block at the beginning of the file:

<Header>
  (main header commands)
</Header>

where the header commands are defined as follows

  • <Description>
   (free-format text)
</Description>
    Provides an optional free-format, text description of the problem. The description must begin on the line after the tag. All characters up to the ending tag are part of the description. 
  • <Analysis>(type)</Analysis>
    Defines type of analysis with with the following options:
    • 0 = Plane strain FEA analysis (NairnFEA only)
    • 1 = Plane stress FEA analysis (NairnFEA only)
    • 2 = Axisymmetric FEA analysis (NairnFEA only)
    • 10 = Plane strain MPM analysis (NairnMPM only)
    • 11 = Plane stress MPM analysis (NairnMPM only)
    • 12 = 3D MPM analysis (NairnMPM only)
    • 13 = Axisymmetric MPM analysis (NairnMPM only)
  • <ConsistentUnits length='(units)' mass='(units)' time='(units)/ConsistentUnits>
    The (units) settings determine metric units for length (which can be km, m, dm cm, mm um (or microns), or nm), mass (which can be kg, g, mg, or ug), and time (which can be s (or sec), ms (or msec), or us). Alternatively, if none of the length, mass, and time attributes are provided, the code will run with generic units for length (L), mass (M), and time (T). The default (if this command is not used) is to use "Legacy Units." If this command is used, you must specify all three attributes or specify none (to use generic units). More details on unit specificatiions are provided here.
  • <Output>(xxxxxx)</Output>
    This optional tag is for NairnFEA only and determines what appears in the output file. The six bytes in the string can be 'Y', 'C', or 'N' and they stdand for:
    • Byte 1: Displacement of each node
    • Byte 2: Nodal forces within each element
    • Byte 3: Nodal stresses within each element
    • Byte 4: Average nodal stresses at each node
    • Byte 5: Force reactivities at nodes with fixed displacement
    • Byte 6: Strain energy of each element and total strain energy
    A 'Y' means that result is fully included in the file. A 'C' means that result is conditionally included only if it is relevant to one of a set of preselected nodes. A 'N' means that result is not included in the file.
  • <Select node='(num)'/>
    Select node (by number) for conditional output of result. This command is rarely used because there are better ways to select nodes for conditional output.
  • <DevelFlag Number='(num)'/>(value)</DevelFlag/>
    Sets a developer flag by (num) (from 0 to 9) to the integer in (value).
